Installation & Setup

yfinance is an open-source Python library that wraps Yahoo Finance's unofficial API, giving you clean programmatic access to historical prices, financial statements, holder data, ESG scores, and more — all for free.

Install

Use pip to install the package. It is recommended to work inside a virtual environment.

⚠ Note
yfinance relies on Yahoo Finance's undocumented API, which can change without notice. Keep the library updated regularly with pip install --upgrade yfinance.
code
# Install via pip
pip install yfinance

# Upgrade to the latest version
pip install --upgrade yfinance

# Optional: install with extra dependencies for faster data parsing
pip install yfinance[nospam,repair]
code
# Standard imports used throughout this guide
import yfinance as yf
import pandas as pd
import json

# Optional: display all rows/columns in DataFrames

pd.set_option('display.max_columns', None)
pd.set_option('display.width', None)
pd.set_option('display.float_format', lambda x: f'{x:,.2f}')

The Ticker Object

The yf.Ticker() class is the primary entry point. Every piece of data you fetch — prices, statements, news — flows through a Ticker instance. You construct one by passing a ticker symbol as a string.

Syntax

ticker = yf.Ticker(ticker, session=None, proxy=None)

Parameters

ParameterTypeDescription
ticker str The ticker symbol for the instrument. Case-insensitive. Required.
session requests.Session Optional custom requests.Session object. Useful for adding custom headers, proxies, or caching via requests_cache. Default: None
proxy str Optional HTTP/HTTPS proxy URL string (e.g., "http://user:pass@host:port"). Default: None

Exchange Suffixes (Indian Markets)

.NSNational Stock Exchange (NSE)
.BOBombay Stock Exchange (BSE)

Common Ticker Formats

AAPLUS Stock (Apple)
BTC-USDCrypto (Bitcoin)
TCS.NSIndian NSE Stock (TCS)
EURUSD=XForex pair
^NSEINifty 50 Index
GC=FGold Futures
code
# ── Basic Ticker instantiation ────────────────────────────────────────────────

# Indian stocks: append .NS (NSE) or .BO (BSE)
ticker_nse = yf.Ticker("TCS.NS")
ticker_bse = yf.Ticker("TCS.BO")

# US stock
apple = yf.Ticker("AAPL")

# Cryptocurrency
bitcoin = yf.Ticker("BTC-USD")

# Indian Index
nifty = yf.Ticker("^NSEI")

# ── Advanced: use requests_cache to avoid redundant network calls ─────────────
import requests_cache

session = requests_cache.CachedSession("yfinance_cache", expire_after=3600)
cached_ticker = yf.Ticker("INFY.NS", session=session)

print(f"Ticker created: {ticker_nse.ticker}")
▶ output
Ticker created: TCS.NS

Stock Info & Metadata

propertydict

The .info property returns a large dictionary with over 100 fields covering company profile, valuation metrics, trading statistics, and contact information. Every access triggers a fresh network request, so always cache it in a variable first.

Key Categories & Fields

FieldTypeDescription
longNamestrFull official company name
symbolstrExchange ticker symbol
sectorstrBroad sector (e.g., "Technology")
industrystrSpecific industry (e.g., "Software—Infrastructure")
currentPricefloatLatest trading price
marketCapintTotal market capitalisation in reporting currency
fiftyTwoWeekHighfloatHighest price in the past 52 weeks
fiftyTwoWeekLowfloatLowest price in the past 52 weeks
fiftyTwoWeekChangePercentfloatPercentage price change over 52 weeks
trailingPEfloatTrailing Price-to-Earnings ratio
forwardPEfloatForward P/E based on analyst estimates
priceToBookfloatPrice-to-Book value ratio
dividendYieldfloatAnnual dividend yield as a decimal (e.g., 0.035 = 3.5%)
betafloatVolatility relative to the broader market
averageVolumeintAverage daily trading volume (10-day)
financialCurrencystrCurrency used in financial statements (e.g., "INR")
longBusinessSummarystrPlain-English business description
⚠ Performance Tip
ticker.info makes a separate HTTP request every single time it is accessed. Always assign info = ticker.info once at the top and reuse the variable throughout your script.

Financial Calendar

propertydict

The .calendar property returns a dictionary of upcoming corporate events for the ticker. It is especially useful for planning around earnings announcements, which can cause significant price volatility.

Typical Keys Returned

KeyDescription
Earnings DateList of expected earnings announcement date(s)
Earnings HighAnalyst high estimate for earnings per share (EPS)
Earnings LowAnalyst low estimate for EPS
Earnings AverageAnalyst consensus (mean) estimate for EPS
Revenue HighAnalyst high estimate for revenue
Revenue LowAnalyst low estimate for revenue
Revenue AverageAnalyst consensus estimate for revenue
Dividend DateNext expected dividend payment date
Ex-Dividend DateDate by which you must own shares to receive the dividend
Note
Calendar data availability is inconsistent for Indian stocks. US stocks tend to have richer calendar coverage. Always check for None before using a value.
code
ticker = yf.Ticker("TCS.NS")
cal = ticker.calendar

print("── Upcoming Financial Events ─────────────────────")
if cal:
    for key, value in cal.items():
        print(f"  {key:<25}: {value}")
else:
    print("  No calendar data available for this ticker.")

# Safe access pattern with fallback
earnings_dates = cal.get("Earnings Date", [])
if earnings_dates:
    print(f"\n  Next Earnings: {earnings_dates[0].strftime('%d %b %Y')}")

Article Dictionary Keys

code
from datetime import datetime

ticker = yf.Ticker("TCS.NS")
news = ticker.news

print(f"Total news articles fetched: {len(news)}")

# ── Inspect raw dictionary of first article ───────────────────────────────────
print("\n── Raw Article Dictionary ────────────────────────")
print(json.dumps(news[0], indent=4))

# ── Pretty-print all articles ─────────────────────────────────────────────────
print("\n── Recent News ───────────────────────────────────")
for i, article in enumerate(news[:5], start=1):

    print(f"    Publisher : {article.get('publisher')}")
    print(f"    Published : {dt_str}")
    print(f"    URL       : {article.get('link')}")

Historical Price Data

methodDataFrame

The .history() method is the workhorse for fetching OHLCV (Open, High, Low, Close, Volume) data. It returns a pandas DataFrame with the date as the index. By default, prices are adjusted for stock splits and dividends (auto_adjust=True).

Syntax

ticker.history(period, interval, start, end, prepost, auto_adjust, back_adjust, repair, actions)

Parameters

ParameterValuesDescription
period "1d" "5d" "1mo" "3mo" "6mo" "1y" "2y" "5y" "10y" "ytd" "max" How far back to fetch data. Cannot be used alongside start/end. Default: "1mo"
interval "1m" "2m" "5m" "15m" "30m" "60m" "90m" "1h" "1d" "5d" "1wk" "1mo" "3mo" Candle granularity. Intraday intervals (1m–90m) are only available for the last 60 days. Default: "1d"
start str or datetime Start date in "YYYY-MM-DD" format. Use with end instead of period.
end str or datetime End date in "YYYY-MM-DD" format (exclusive). Defaults to today.
prepost bool Include pre-market and after-hours data (US markets only). Default: False
auto_adjust bool Automatically adjust OHLC prices for splits and dividends. Default: True
actions bool Include Dividends and Stock Splits columns in the returned DataFrame. Default: False
repair bool Attempt to detect and fix bad price data (e.g., 100x errors). Default: False
Returns → DataFrame
Columns: Open, High, Low, Close, Volume (and optionally Dividends, Stock Splits). Index is a timezone-aware DatetimeIndex.
code
wipro = yf.Ticker("WIPRO.NS")

# ── 1. Intraday: today's 1-minute candles ─────────────────────────────────────
# Note: 1m data only available for the last 7 days
today_df = wipro.history(period="1d", interval="1m")
current_price = today_df["Close"].iloc[-1] if not today_df.empty else "N/A"
print(f"Current Wipro Price (NSE) : ₹{current_price:.2f}")

# ── 2. Last 30 days, daily candles ───────────────────────────────────────────
monthly_df = wipro.history(period="1mo", interval="1d")
print("\n── Last 5 Days High / Low ────────────────────────")
print(monthly_df[["High", "Low"]].tail())

# ── 3. Custom date range ─────────────────────────────────────────────────────
custom_df = wipro.history(start="2024-01-01", end="2024-12-31", interval="1wk")
print("\n── Weekly Data for 2024 (first 5 rows) ───────────")
print(custom_df[["Open", "High", "Low", "Close", "Volume"]].head())

# ── 4. Max history including corporate actions ────────────────────────────────
max_df = wipro.history(period="max", actions=True)
print(f"\n── Max Available History ─────────────────────────")
print(f"From: {max_df.index.min().date()}  To: {max_df.index.max().date()}")
print(f"Total candles: {len(max_df)}")

# ── 5. Useful derived calculations ───────────────────────────────────────────
yearly_df = wipro.history(period="1y")
yearly_df["Daily Return %"] = yearly_df["Close"].pct_change() * 100
yearly_df["MA_20"] = yearly_df["Close"].rolling(20).mean()   # 20-day moving avg
yearly_df["MA_50"] = yearly_df["Close"].rolling(50).mean()   # 50-day moving avg

print("\n── 1Y Stats ──────────────────────────────────────")
print(f"Avg Daily Return: {yearly_df['Daily Return %'].mean():.3f}%")
print(f"Volatility (std): {yearly_df['Daily Return %'].std():.3f}%")
print(f"Best Day        : {yearly_df['Daily Return %'].max():.2f}%")
print(f"Worst Day       : {yearly_df['Daily Return %'].min():.2f}%")

Financial Statements

propertyDataFrame

yfinance exposes all three core financial statements for fundamental analysis. Each statement has an annual and a quarterly version. Rows represent line items (metrics), and columns represent reporting periods (most recent first).

Income Statement

Shows revenue, expenses, and profitability over a period.

.financialsAnnual — last 4 years
.quarterly_financialsQuarterly — last 4 quarters

Key rows include: Total Revenue, Gross Profit, EBITDA, Operating Income, Net Income, Basic EPS, Diluted EPS.

Balance Sheet

A point-in-time snapshot of assets, liabilities, and shareholders' equity.

.balance_sheetAnnual
.quarterly_balance_sheetQuarterly

Key rows: Total Assets, Total Debt, Cash And Cash Equivalents, Stockholders Equity, Current Liabilities.

Cash Flow Statement

Tracks actual cash movement — operating, investing, and financing activities.

.cashflowAnnual
.quarterly_cashflowQuarterly

Key rows: Free Cash Flow, Operating Cash Flow, Capital Expenditure, Repurchase Of Capital Stock.

code
tata_steel = yf.Ticker("TATASTEEL.NS")

# ── Income Statement ──────────────────────────────────────────────────────────
income_stmt   = tata_steel.financials
q_income_stmt = tata_steel.quarterly_financials

print("── Annual Income Statement (last 2 years) ────────")
print(income_stmt.iloc[:, :2])  # .iloc[:, :2] → all rows, first 2 columns

# Access a specific row
if "Total Revenue" in income_stmt.index:
    revenue = income_stmt.loc["Total Revenue"].iloc[0]
    print(f"\nLatest Annual Revenue : ₹{revenue / 1e9:.2f} Billion")

# ── Balance Sheet ─────────────────────────────────────────────────────────────
annual_bs    = tata_steel.balance_sheet
quarterly_bs = tata_steel.quarterly_balance_sheet

print("\n── Quarterly Balance Sheet (most recent quarter) ─")
print(quarterly_bs.iloc[:, 0])  # First column = most recent quarter

# Debt-to-Equity ratio
try:
    total_debt   = quarterly_bs.loc["Total Debt"].iloc[0]
    total_equity = quarterly_bs.loc["Stockholders Equity"].iloc[0]
    dte = total_debt / total_equity
    print(f"\nDebt-to-Equity Ratio : {dte:.2f}")
except (KeyError, ZeroDivisionError):
    print("  Could not compute D/E ratio.")

# ── Cash Flow Statement ───────────────────────────────────────────────────────
cash_flow = tata_steel.cashflow

print("\n── Annual Cash Flow (top 5 rows) ─────────────────")
print(cash_flow.head(5))

if "Free Cash Flow" in cash_flow.index:
    fcf = cash_flow.loc["Free Cash Flow"].iloc[0]
    print(f"\nLatest Free Cash Flow : ₹{fcf / 1e9:.2f} Billion")

Corporate Actions

propertyDataFrame

Corporate actions are events initiated by a public company that materially affect the stock. The two most important are Dividends (cash distributions to shareholders) and Stock Splits (increasing share count, reducing price per share proportionally).

Properties Available

PropertyReturnsDescription
.actionsDataFrameCombined view of both Dividends and Splits, indexed by date
.dividendsSeriesHistorical dividend amounts per share, indexed by ex-dividend date
.splitsSeriesHistorical stock split ratios (e.g., 2.0 = 2-for-1 split), indexed by date
.capital_gainsSeriesCapital gains distributions (relevant for ETFs/mutual funds)
code
itc = yf.Ticker("ITC.NS")
info = itc.info

# ── Fetch all corporate actions ───────────────────────────────────────────────
actions = itc.actions     # Combined DataFrame: Dividends + Splits
divs    = itc.dividends   # Series: dividend per share by ex-date
splits  = itc.splits      # Series: split ratio by date

print(f"── Corporate Actions: {info.get('longName')} ──")

if not actions.empty:
    print("\nRecent Corporate Actions (last 5):")
    print(actions.tail(5))

# ── Dividend Analysis ─────────────────────────────────────────────────────────
if not divs.empty:
    currency = info.get('currency', 'INR')
    print(f"\n── Dividend Summary ──────────────────────────────")
    print(f"Total dividends paid   : {len(divs)}")
    print(f"Last dividend amount   : {divs.iloc[-1]:.2f} {currency}")
    print(f"Avg annual dividend    : {divs.resample('YE').sum().mean():.2f} {currency}")
    print("\nLast 5 Dividends:")
    print(divs.tail(5))

# ── Stock Splits ──────────────────────────────────────────────────────────────
if not splits.empty:
    print("\n── Historical Stock Splits ───────────────────────")
    print(splits)
else:
    print("\nNo historical stock splits found.")

Analyst Recommendations & Price Targets

propertyDataFrame / dict

yfinance exposes analyst opinion data — both consensus price targets and a breakdown of Buy/Hold/Sell ratings from Wall Street analysts and brokers.

Price Targets (from .info)

KeyDescription
targetMeanPriceAnalyst consensus (average) 12-month price target
targetHighPriceMost optimistic analyst price target
targetLowPriceMost pessimistic analyst price target
targetMedianPriceMedian price target across all analysts
recommendationKeyConsensus recommendation string: "strong_buy", "buy", "hold", "sell"
numberOfAnalystOpinionsTotal number of analysts contributing to consensus

Properties

PropertyReturnsDescription
.recommendations_summaryDataFrameCounts of each rating grade for the current and prior periods
.recommendationsDataFrameDetailed historical recommendations from individual firms
.analyst_price_targetsDataFrameCurrent, low, high, and mean price targets
Coverage Note
Analyst coverage data is sparse for mid-cap and small-cap Indian stocks. US-listed stocks generally have the richest coverage. Always check for None before use.
code
sbi = yf.Ticker("SBIN.NS")

# ── Recommendation Summary ────────────────────────────────────────────────────

# ── Historical Recommendations (firm-level) ───────────────────────────────────

if recs is not None and not recs.empty:
    print("\n── Latest Firm-Level Recommendations ─────────────")

Ownership & Holders

propertyDataFrame

Understanding who holds a stock — promoters, institutions, or retail investors — is crucial for gauging conviction and stability. yfinance exposes three layers of ownership data, updated based on regulatory filings (e.g., SEBI in India, SEC in the US).

Properties

PropertyReturnsDescription
.major_holdersDataFrameHigh-level percentage summary: insider %, institutions %, float outstanding, etc.
.institutional_holdersDataFrameTop institutional investors (FIIs, pension funds, asset managers) with share counts and value
.mutualfund_holdersDataFrameTop mutual funds invested in the stock with share counts and value
.insider_transactionsDataFrameRecent insider buying/selling activity
.insider_roster_holdersDataFrameNamed insiders and their current holdings

Major Holders Row Labels

Row LabelMeaning
insidersPercentHeldFraction of shares held by company insiders (promoters)
institutionsPercentHeldFraction held by all institutional investors
institutionsFloatPercentHeldFraction of the public float held by institutions

Candlestick & OHLC Charts

Candlestick charts encode four data points per period — Open, High, Low, Close — in a single visual element. They are the standard chart type for reading price action and detecting chart patterns.

Anatomy of a Candlestick

  • Body — the rectangle between Open and Close. Green (hollow) = close > open (bullish). Red (filled) = close < open (bearish).
  • Upper Wick — line from body top to the High. Shows rejected upward moves.
  • Lower Wick — line from body bottom to the Low. Shows rejected downward moves.

Technical Indicators & Oscillator Charts

Technical indicators transform raw OHLCV data into signals about momentum, trend strength, volatility, and overbought/oversold conditions. All of the below are computed manually using pandas — no extra library required (though pandas-ta or TA-Lib can automate this at scale).

Indicators Covered

RSI — Relative Strength Index

Measures momentum on a 0–100 scale. Traditionally, RSI > 70 = overbought (potential sell), RSI < 30 = oversold (potential buy). Use a 14-period window by convention.

Formula: RSI = 100 − 100 / (1 + RS) where RS = avg_gain / avg_loss over N periods.

MACD — Moving Average Convergence Divergence

Shows the relationship between two EMAs. The MACD line (EMA12 − EMA26) crossing above the Signal line (EMA9 of MACD) is a bullish signal. The Histogram (MACD − Signal) shows momentum strength.

Bollinger Bands

A volatility envelope around a 20-day SMA. Upper/Lower bands are set at ±2 standard deviations. Price touching the upper band suggests overbought; lower band suggests oversold. A squeeze (bands narrowing) precedes a volatility breakout.

ATR — Average True Range

Measures volatility as the average of the true range (max of high−low, |high−prev_close|, |low−prev_close|) over 14 periods. Higher ATR = more volatile market; useful for setting stop-loss distances.

Library shortcut
Install pandas-ta (pip install pandas-ta) to compute all indicators with one line: df.ta.rsi(length=14), df.ta.macd(), df.ta.bbands(), etc.

# ── Helper functions — compute indicators from scratch ────────────────────────

def compute_rsi(series, period=14):
    delta = series.diff()
    gain  = delta.clip(lower=0).ewm(com=period - 1, adjust=False).mean()
    loss  = (-delta.clip(upper=0)).ewm(com=period - 1, adjust=False).mean()
    rs    = gain / loss
    return 100 - (100 / (1 + rs))

def compute_macd(series, fast=12, slow=26, signal=9):
    ema_fast   = series.ewm(span=fast,   adjust=False).mean()
    ema_slow   = series.ewm(span=slow,   adjust=False).mean()
    macd_line  = ema_fast - ema_slow
    signal_line= macd_line.ewm(span=signal, adjust=False).mean()
    histogram  = macd_line - signal_line
    return macd_line, signal_line, histogram

def compute_bollinger(series, window=20, num_std=2):
    sma        = series.rolling(window).mean()
    std        = series.rolling(window).std()
    upper      = sma + num_std * std
    lower      = sma - num_std * std
    return upper, sma, lower

def compute_atr(df, period=14):
    hl  = df["High"] - df["Low"]
    hcp = (df["High"] - df["Close"].shift()).abs()
    lcp = (df["Low"]  - df["Close"].shift()).abs()
    tr  = pd.concat([hl, hcp, lcp], axis=1).max(axis=1)
    return tr.ewm(span=period, adjust=False).mean()

Chart 7 — Stochastic Oscillator + OBV

Two more momentum/volume tools that complement RSI and MACD:

  • Stochastic Oscillator (%K, %D) — Compares a closing price to its price range over N periods (default 14). %K = raw oscillator, %D = 3-period SMA of %K. Crossovers above 80 / below 20 signal reversals.
  • OBV — On-Balance Volume — Cumulative volume that adds volume on up-days and subtracts on down-days. OBV rising with price = healthy trend. OBV diverging from price = early reversal warning.
code
def compute_stochastic(df, k_period=14, d_period=3):
    low_min  = df["Low"].rolling(k_period).min()
    high_max = df["High"].rolling(k_period).max()
    pct_k    = 100 * (df["Close"] - low_min) / (high_max - low_min)
    pct_d    = pct_k.rolling(d_period).mean()
    return pct_k, pct_d

def compute_obv(df):
    direction = df["Close"].diff().apply(lambda x: 1 if x > 0 else (-1 if x < 0 else 0))
    return (direction * df["Volume"]).cumsum()

df["%K"], df["%D"] = compute_stochastic(df)
df["OBV"]          = compute_obv(df)
