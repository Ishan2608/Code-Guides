00 — Foundations
What is a Continuous RV?

A random variable X is continuous when it can take any value in an interval — uncountably many possible outcomes. Think measurements: height, time, temperature. No single point has positive probability.

PDF — Probability Density Function

The PDF gives density, not probability. P(X = x) = 0 always. Probability only exists over intervals — it is the area under the curve.

P(a ≤ X ≤ b) = ∫ₐᵇ f(x) dx
Two PDF Rules (always hold)

Every valid PDF must satisfy both axioms — integrals replace the sums used in PMFs.

① f(x) ≥ 0 for all x
② ∫₋∞^∞ f(x) dx = 1
CDF — Cumulative Distribution Function

F(x) = P(X ≤ x). Always non-decreasing; F(−∞)=0 and F(+∞)=1. The PDF is the derivative of the CDF.

F(x) = ∫₋∞ˣ f(t) dt
f(x) = dF(x)/dx
Key Moments (via integration)

Expected value and variance are defined by integrating over the full support — the continuous analogues of weighted sums.

E[X] = ∫ x·f(x) dx
Var(X) = E[X²] − (E[X])²
Standardization

Any continuous RV can be standardized. The result always has mean 0 and variance 1, enabling table lookups and comparisons across different distributions.

Z = (X − μ) / σ
01 — Uniform
U(a,b) Uniform Distribution Maximum Uncertainty
Simple Intuition

"Pick a random point anywhere on a line segment. Every location is equally likely — no part of the interval is more probable than any other of the same width."

Technical Definition

All values in [a, b] share the same constant density 1/(b−a). It is the maximum entropy distribution over a bounded interval — it encodes zero prior preference for any sub-region. The simplest continuous distribution and the starting point for simulation.

Conditions
  • ▸Outcome is bounded to a finite interval [a, b]
  • ▸All sub-intervals of equal width are equally probable
Real-World Use Cases

Random number generation in software. Rounding errors in numerical computation. Arrival time when no schedule is known. Uninformative Bayesian priors. Simulation seeding via the Probability Integral Transform.

PDF & CDF
f(x) = 1 / (b − a), x ∈ [a, b]
f(x) = 0, otherwise

F(x) = (x − a) / (b − a), x ∈ [a, b]
// Flat PDF — the rectangle shape
// CDF is a straight diagonal line
Parameters & Moments
Parameters
a, b ∈ ℝ, a < b
Support
x ∈ [a, b]
Mean E[X]
(a + b) / 2
Variance
(b − a)² / 12
Median
(a + b) / 2
Skewness
0 (symmetric)
Probability Integral Transform: If X ~ U(0,1) and F is any valid CDF, then F⁻¹(X) follows the distribution with CDF F. This is how simulation software generates samples from any arbitrary distribution using only a uniform RNG.

02 — Normal (Gaussian)
N(μ, σ²) Normal Distribution The Bell Curve
Simple Intuition

"Measure anything that results from many small, independent influences added together. The total clusters around a center and tapers off symmetrically in both directions — a bell curve."

Technical Definition

The most important distribution in statistics. Its bell shape arises via the Central Limit Theorem: the sample mean of any large i.i.d. sample is approximately Normal, regardless of the original distribution. μ controls location; σ² controls spread.

The 68–95–99.7 Rule
  • ▸68% of data lies within μ ± 1σ
  • ▸95% of data lies within μ ± 2σ
  • ▸99.7% of data lies within μ ± 3σ
Real-World Use Cases

Human height and weight. Measurement errors in science. IQ scores. Financial return approximations. Residuals in linear regression. Any sum of many independent random effects.

PDF & Standard Form
f(x) = 1/(σ√2π) · exp(−(x−μ)²/2σ²)

Standard Z = (X−μ)/σ ~ N(0,1)
f(z) = 1/√(2π) · exp(−z²/2)
// No closed-form CDF — use z-tables or erf()
// Φ(z) = P(Z ≤ z) is the standard normal CDF

Support
x ∈ (−∞, +∞)
Mean E[X]
μ
Variance
σ²
Skewness
0 (perfectly symmetric)
Excess Kurtosis
0 (the reference point)
Reproductive property: If X~N(μ₁,σ₁²) and Y~N(μ₂,σ₂²) independently, then X+Y~N(μ₁+μ₂, σ₁²+σ₂²). Sums of independent normals are always normal.

03 — Exponential
Exp(λ) Exponential Distribution Time Between Events
Simple Intuition

"Customers arrive at a rate of λ per hour. How long do you wait for the next one? That waiting time is Exponential — short waits are most likely, long waits are rare but possible."

Technical Definition

The continuous analogue of the Geometric distribution. X measures time between consecutive events in a Poisson process with rate λ. The PDF decays as a pure exponential. λ is the rate (events per unit time), so 1/λ is the average wait.

Memoryless Property
  • ▸P(X > s+t | X > s) = P(X > t)
  • ▸Past waiting time gives no information about future wait
  • ▸The only continuous distribution with this property
Real-World Use Cases

Time until next server request. Lifetime of electronic components before failure. Time between radioactive decay events. Inter-arrival times in a queueing system. Duration of phone calls.

PDF & CDF
f(x) = λ · e^(−λx), x ≥ 0
f(x) = 0, x < 0

F(x) = 1 − e^(−λx), x ≥ 0
// λ = rate (events per unit time)
// Mean wait = 1/λ (e.g. λ=2 → avg wait = 0.5)
Parameters & Moments
Parameter
λ > 0 (rate)
Support
x ∈ [0, +∞)
Mean E[X]
1 / λ
Variance
1 / λ²
Median
ln(2) / λ ≈ 0.693/λ
Skewness
2 (always right-skewed)
Poisson connection: If events follow Poisson(λ) in time, then inter-arrival times follow Exp(λ). One counts events; the other measures gaps. Two views of the same underlying Poisson process.

04 — Gamma
Gamma(α, β) Gamma Distribution Generalised Waiting Time
Simple Intuition

"You need to wait for not just 1 event, but α events in a row. The total waiting time — the sum of α independent exponential waits — follows a Gamma distribution."

Technical Definition

X is the sum of α independent Exp(1/β) random variables. α (shape) controls how many exponential stages are summed; β (scale) stretches the time axis. The Gamma function Γ(α) generalises the factorial: Γ(n) = (n−1)! for positive integers. When α=1, it collapses to Exponential.

Special Cases
  • ▸α = 1 → Exponential(1/β)
  • ▸α = k/2, β = 2 → Chi-Squared(k)
  • ▸Integer α → Erlang distribution (used in queuing)
Real-World Use Cases

Total service time for multiple tasks. Insurance claim sizes. Time until α-th failure in a reliability system. Rainfall totals. Bayesian prior for a Poisson rate λ (conjugate prior).

PDF
f(x) = x^(α−1) · e^(−x/β)
       ─────────────────
       β^α · Γ(α)

x > 0, α > 0, β > 0
// Γ(α) = ∫₀^∞ t^(α−1)e^(−t)dt (generalised factorial)
// α = shape, β = scale
Parameters & Moments
Parameters
α > 0 (shape), β > 0 (scale)
Support
x ∈ (0, +∞)
Mean E[X]
αβ
Variance
αβ²
Mode
(α − 1)β, α ≥ 1
Skewness
2 / √α
Additive property: If X~Gamma(α₁,β) and Y~Gamma(α₂,β) independently, then X+Y~Gamma(α₁+α₂, β). Shapes add when the scale is shared — mirrors the Poisson additive property.

05 — Beta
Beta(α, β) Beta Distribution Modelling Probabilities
Simple Intuition

"You want a distribution that only lives between 0 and 1 — for modelling a probability, proportion, or rate. Beta is the answer. Adjust α and β to get almost any shape you need."

Technical Definition

Defined on [0,1], shaped by two positive parameters. It is the conjugate prior for the Binomial in Bayesian statistics: start with Beta(α,β) as your prior belief about a probability p; observe k successes in n trials; posterior is Beta(α+k, β+n−k). No re-computation needed — just update the counts.

Shape Behaviour
  • ▸α = β = 1 → Uniform(0,1)
  • ▸α > β → left-skewed (mass toward 1)
  • ▸α < β → right-skewed (mass toward 0)
  • ▸α = β → symmetric about 0.5
  • ▸α, β < 1 → U-shaped (bimodal at endpoints)
Real-World Use Cases

Bayesian posterior for a probability. Proportion of defective items. Click-through rates in A/B testing. Project completion fractions. Any measurement naturally bounded in [0,1].

PDF
f(x) = x^(α−1) · (1−x)^(β−1)
       ─────────────────────
       B(α, β)

x ∈ [0,1], α > 0, β > 0
// B(α,β) = Γ(α)Γ(β)/Γ(α+β) (Beta function)
// Normalising constant ensuring ∫f=1
Parameters & Moments
Parameters
α > 0, β > 0
Support
x ∈ [0, 1]
Mean E[X]
α / (α + β)
Variance
αβ / [(α+β)²(α+β+1)]
Mode
(α−1)/(α+β−2), α,β > 1
Special
Beta(1,1) = Uniform(0,1)
Bayesian update: Prior Beta(α,β) + k successes + (n−k) failures → Posterior Beta(α+k, β+n−k). The parameters act as pseudo-counts of prior experience with successes and failures.

06 — Student's t
t(ν) Student's t-Distribution Small-Sample Inference
Simple Intuition

"It looks like the Normal distribution, but with heavier tails — because your σ is estimated from a small sample, not known exactly. Smaller samples mean heavier tails and more uncertainty."

Technical Definition

Arises as Z/√(χ²/ν) — the ratio of a standard Normal to the square root of a scaled Chi-Squared. It is the exact distribution of the t-statistic (x̄−μ)/(s/√n) when sampling from a Normal population with unknown σ. As ν→∞, t(ν)→N(0,1) exactly.

Degrees of Freedom ν
  • ▸One-sample t-test: ν = n − 1
  • ▸Two-sample t-test: ν ≈ n₁ + n₂ − 2
  • ▸Small ν → very heavy tails → more conservative CIs
  • ▸ν ≥ 30: practically indistinguishable from Normal
Real-World Use Cases

Hypothesis tests for means when σ is unknown. Confidence intervals from small samples. Regression coefficient significance tests. A/B testing with small sample sizes.

PDF
f(t) = Γ((ν+1)/2)
      ───────────────────────────────
      √(νπ)·Γ(ν/2)·(1 + t²/ν)^((ν+1)/2)
// ν = degrees of freedom
// Symmetric about 0; heavier tails than Normal
Parameters & Moments
Parameter
ν > 0 (df)
Support
t ∈ (−∞, +∞)
Mean E[X]
0 (ν > 1)
Variance
ν/(ν−2) (ν > 2)
Skewness
0 (symmetric)
Excess Kurtosis
6/(ν−4) (ν > 4)
Construction: If Z~N(0,1) and V~χ²(ν) are independent, then T = Z/√(V/ν) ~ t(ν). This is why df matters — it controls how noisy the estimated standard deviation is in the denominator.

07 — Chi-Squared
χ²(k) Chi-Squared Distribution Sum of Squared Normals
Simple Intuition

"Take k independent standard Normal values. Square each one. Add them all up. That total follows a Chi-Squared distribution with k degrees of freedom."

Technical Definition

If Z₁,…,Zₖ are i.i.d. N(0,1), then X = Z₁²+…+Zₖ² ~ χ²(k). It is a special case of Gamma(k/2, 2). Always non-negative and right-skewed, becoming more symmetric and Normal-like as k increases (by CLT).

Where It Appears
  • ▸Goodness-of-fit test: χ² = Σ(O−E)²/E
  • ▸Test of independence (contingency tables)
  • ▸Confidence interval for variance: (n−1)s²/σ² ~ χ²(n−1)
  • ▸Building block for t and F distributions
Real-World Use Cases

Testing if observed categorical frequencies match a theoretical distribution. Testing independence between two categorical variables. Estimating population variance from a sample.

PDF
f(x) = x^(k/2−1) · e^(−x/2)
       ──────────────────────
       2^(k/2) · Γ(k/2)

x > 0
// Special case: Gamma(k/2, 2)
// k = degrees of freedom (positive integer)
Parameters & Moments
Parameter
k ∈ ℤ⁺ (df)
Support
x ∈ (0, +∞)
Mean E[X]
k
Variance
2k
Mode
max(k − 2, 0)
Skewness
√(8/k) → 0 as k → ∞
Additive property: If X~χ²(k₁) and Y~χ²(k₂) independently, then X+Y~χ²(k₁+k₂). Degrees of freedom simply add — just like variance adds for independent Normal sums.

08 — F-Distribution
F(d₁, d₂) F-Distribution Ratio of Variances
Simple Intuition

"Divide two independent Chi-Squared variables, each scaled by its degrees of freedom. The ratio follows an F-distribution — and a large F tells you one source of variance is much bigger than the other."

Technical Definition

If U~χ²(d₁) and V~χ²(d₂) independently, then F = (U/d₁)/(V/d₂) ~ F(d₁,d₂). Always non-negative and right-skewed. In ANOVA, the F-statistic compares between-group to within-group variance — a large F means groups differ beyond chance.

Where It Appears
  • ▸ANOVA overall F-test: MSB / MSW
  • ▸Regression overall significance: MSR / MSE
  • ▸Testing equality of two population variances
  • ▸Levene's and Bartlett's variance tests
Real-World Use Cases

ANOVA: do three teaching methods produce different mean scores? Regression: does the model explain significantly more variance than chance alone? Comparing process variability across two factories.

PDF
f(x) = √((d₁x)^d₁·d₂^d₂/(d₁x+d₂)^(d₁+d₂))
       ─────────────────────────────────────
       x · B(d₁/2, d₂/2)

x > 0
// d₁ = numerator df, d₂ = denominator df
// 1/F ~ F(d₂, d₁) (reciprocal is also F)
Parameters & Moments
Parameters
d₁ > 0, d₂ > 0
Support
x ∈ (0, +∞)
Mean E[X]
d₂/(d₂−2) (d₂ > 2)
Mode
d₁(d₂−2)/[d₂(d₁+2)] (d₁>2)
Relation
t²(ν) ~ F(1, ν)
Relation
1/F ~ F(d₂, d₁)
t–F connection: If T~t(ν), then T²~F(1,ν). A two-tailed t-test with ν df is equivalent to an F-test with (1,ν) df. ANOVA with two groups reduces exactly to a two-sample t-test.

09 — Log-Normal
LogN(μ, σ²) Log-Normal Distribution Multiplicative Growth
Simple Intuition

"If taking the logarithm of X gives you a Normal distribution, then X itself is Log-Normal. It models quantities that grow multiplicatively — like prices, incomes, or bacterial populations."

Technical Definition

If Y~N(μ,σ²), then X = eʸ ~ LogNormal(μ,σ²). Equivalently, ln(X)~N(μ,σ²). The multiplicative CLT applies: when many independent positive factors are multiplied together, their product tends toward Log-Normal. Always strictly positive and right-skewed.

Key Properties
  • ▸Strictly positive: X > 0 always
  • ▸Right-skewed regardless of parameters
  • ▸μ, σ² are parameters of ln(X), not of X itself
  • ▸Mean > Median > Mode always (skew signature)
Real-World Use Cases

Stock prices and financial asset returns. Income and wealth distributions. City and website population sizes. Biological concentrations (blood levels, cell sizes). Internet traffic and file sizes. Insurance claim amounts.

PDF
f(x) = exp(−(ln x − μ)² / 2σ²)
       ─────────────────────────
       x · σ · √(2π)

x > 0
// μ, σ are the mean and std dev of ln(X)
// E[X] = e^(μ+σ²/2) ← not e^μ
Parameters & Moments
Parameters
μ ∈ ℝ, σ² > 0 (of ln X)
Support
x ∈ (0, +∞)
Mean E[X]
e^(μ + σ²/2)
Variance
(e^σ²−1) · e^(2μ+σ²)
Median
e^μ
Mode
e^(μ − σ²)
Mean > Median > Mode always. Because of the right skew, these three measures are always strictly ordered this way. Seeing this ordering in count or measurement data is a strong signal the underlying process may be log-normal.
