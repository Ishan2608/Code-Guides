Chapter — Unsupervised Learning
Anomaly Detection
One-word definition Outliers
Anomaly Detection is an unsupervised machine learning discipline that identifies rare observations deviating significantly from established patterns in data, enabling automated discovery of fraud, faults, intrusions, and other critical irregular events without requiring labelled examples.
Why It Matters — Real-World Applications
Finance
Credit card fraud detection. Unusual transaction amounts or locations.
Cybersecurity
Network intrusion detection. Abnormal login patterns or traffic spikes.
Manufacturing
Predictive maintenance. Sensor readings outside normal operating range.
Healthcare
Rare disease detection. Patient vitals deviating from expected baselines.
Three Types of Anomalies
Type A — Point Anomaly
Single Outlier
A single data instance is anomalous relative to the rest. Most common type.
Example: One transaction of $50,000 among average $200 purchases.
Type B — Contextual Anomaly
Context-Dependent
A point is normal globally but anomalous in a specific context (e.g., time, location).
Example: 25°C is normal in summer but anomalous in December in Moscow.
Type C — Collective Anomaly
Group Outlier
Individual points are normal, but their collective sequence or pattern is anomalous.
Example: Normal ECG values in an abnormal temporal pattern indicating arrhythmia.

Core Metrics & Terms — Shared Across All Methods
Anomaly Score
A numerical measure of how anomalous a point is. Higher score (or further from 0) = more likely to be an outlier. Each algorithm defines its own score.
score(x) → high = anomaly
Contamination Rate
The expected proportion of anomalies in your dataset. A key hyperparameter for most algorithms. Setting it correctly is crucial — too high flags too many normals.
contamination ∈ (0, 0.5)
Precision / Recall
Anomaly detection is a detection problem. Precision = of flagged anomalies, how many were real? Recall = of real anomalies, how many did we catch?
F1 = 2·(P·R)/(P+R)
Inlier vs Outlier: An inlier is a normal data point that fits the expected pattern. An outlier (anomaly) is a point that deviates significantly. The decision boundary between them is drawn by a threshold on the anomaly score.
Supervised vs Unsupervised: Most real anomaly detection is unsupervised — labels for anomalies are unavailable or expensive. Algorithms must learn "normal" from unlabelled data, then flag deviations.
Four Types of Anomaly Detection Methods
Type 01
Statistical
Model data distribution. Anomalies are points with low probability under that model.
→ Z-Score, IQR, Grubbs
Type 02
Isolation-Based
Anomalies are easier to isolate in feature space than normal points. Exploit this.
→ Isolation Forest, Extended IF
Type 03
Density-Based
Anomalies lie in low-density regions. Compare a point's local density to its neighbours.
→ LOF, LOCI, DBSCAN
Type 04
Boundary-Based
Learn a boundary enclosing normal data. Anything outside is an anomaly.
→ One-Class SVM, SVDD
Type 01 · Statistical Method
Z-Score Detection
Also covers IQR (Interquartile Range) · Parametric approach
Z-Score detection assumes data follows a Gaussian distribution, measuring how many standard deviations each point lies from the mean — flagging points beyond a defined threshold as statistically improbable and therefore anomalous.
⚡ Core Assumption: Normal data is Gaussian (bell-curve distributed). Under a true normal distribution, 99.7% of data falls within ±3σ of the mean. Any point outside this range has less than 0.3% probability of being normal — making it a strong anomaly candidate.
Key Terms
Mean (μ): The average of all data points. The centre of the Gaussian distribution around which "normal" is defined.
Standard Deviation (σ): Measure of spread. How far data typically varies from the mean. A small σ = tight cluster; large σ = wide spread.
Z-Score: The standardised distance of a point from the mean, measured in standard deviation units. Z = (x − μ) / σ. Dimensionless — works across different feature scales.
Threshold (t): The absolute Z-Score cutoff. Typically |Z| > 2.5 (moderate) or |Z| > 3 (strict). Points exceeding this are labelled anomalies.
IQR (Interquartile Range): A non-parametric alternative. IQR = Q3 − Q1. Anomalies = points below Q1 − 1.5·IQR or above Q3 + 1.5·IQR. Robust to outliers in the training set itself.
Modified Z-Score: Uses median and MAD (Median Absolute Deviation) instead of mean and std, making it more robust to the very outliers it tries to detect.
Algorithm Steps — Z-Score
01
Compute the mean (μ)
Calculate the average of the feature across all training data points.
02
Compute standard deviation (σ)
Measure how spread the data is around the mean. Low σ = tight normal cluster.
03
Compute Z-Score for each point
Z = (x − μ) / σ. Positive = above mean, negative = below. Magnitude = distance in σ units.
04
Apply threshold
Flag any point where |Z| > threshold (e.g., 3). These fall beyond the 3σ boundary.
05
For multivariate data: apply per feature or use Mahalanobis Distance
Mahalanobis extends Z-Score to multiple correlated features simultaneously, accounting for covariance structure.
Key Limitation: Z-Score assumes Gaussian distribution. On skewed, multimodal, or heavy-tailed data it will produce high false-positive or false-negative rates. Always check your distribution first.
Best for: Unimodal, roughly normal features. Time-series monitoring with known baselines. Fast, interpretable, no training needed.
Code Syntax
zscore.py
# ── 1. Import ─────────────────────────────
import numpy as np
from scipy import stats
import pandas as pd

# ── Z-Score method ────────────────────────
# ── 2. Compute mean & std ─────────────────
mu    = X.mean()
sigma = X.std()

# ── 3. Compute Z-Scores ───────────────────
z_scores = np.abs((X - mu) / sigma)

# ── 4. Flag anomalies ─────────────────────
threshold  = 3
anomalies  = X[z_scores > threshold]

# ── Scipy shortcut ────────────────────────
z         = np.abs(stats.zscore(X))
anomalies = X[(z > 3).any(axis=1)]

# ── IQR Method (non-parametric) ───────────
Q1  = X.quantile(0.25)
Q3  = X.quantile(0.75)
IQR = Q3 - Q1

lower     = Q1 - 1.5 * IQR
upper     = Q3 + 1.5 * IQR
anomalies = X[(X < lower) | (X > upper)]

Type 02 · Isolation-Based Method
Isolation Forest
Liu, Ting & Zhou, 2008 · sklearn.ensemble.IsolationForest
Isolation Forest exploits the observation that anomalies are few and different — they require far fewer random partitions to isolate than normal points, producing shorter path lengths in randomly constructed trees that directly encode anomaly scores.
⚡ Core Insight: Normal points live in dense, clustered regions — it takes many random splits to isolate them. Anomalies are sparse and distinct — they get isolated with just a few splits. Average path length in a forest of random trees directly measures how hard it is to isolate a point. Short path = anomaly.
Key Terms
Isolation Tree (iTree): A randomly built binary decision tree. At each node, a random feature is selected and a random split point is chosen within that feature's range. No labels, no optimisation criterion.
Path Length h(x): The number of edges traversed from root to the leaf node where point x is isolated. Short path = anomaly. Long path = normal.
Anomaly Score s(x,n): Normalised score between 0 and 1. s → 1 = anomaly. s → 0.5 = normal. s → 0 = very normal. Derived from average path length across all trees.
n_estimators: Number of isolation trees in the forest (default: 100). More trees = more stable scores. Trade-off: computation time.
max_samples: Number of data points sampled to build each tree (default: 256). Surprisingly small sub-samples work well — anomalies still separate quickly.
contamination: Expected fraction of anomalies. Used to set the decision threshold on s(x). If unknown, start with 0.1 (10%) and tune from there.
Algorithm Steps
01
Build T = n_estimators isolation trees
For each tree: sample max_samples points. Recursively split by random feature + random threshold until each point is isolated or max_depth is reached.
02
Compute path length h(x) for each point
Pass each point through all T trees. Record the depth at which it reaches a leaf node in each tree.
03
Average path length across all trees
E[h(x)] = average depth across all T trees. This expectation is a stable measure of isolation difficulty.
04
Compute anomaly score s(x, n)
s(x,n) = 2^(−E[h(x)] / c(n)), where c(n) is the expected path length of an unsuccessful BST search — normalises scores across sample sizes.
05
Apply contamination threshold → label predictions
Sort points by score. Top contamination% with highest scores are labelled as anomalies (−1). The rest are normal (+1).
Limitation: Struggles with anomalies that are similar to each other (clustered anomalies) and with high-dimensional data where random splits become less discriminative. Extended Isolation Forest (EIF) addresses this.
Code Syntax
isolation_forest.py
# ── 1. Import ─────────────────────────────
from sklearn.ensemble import IsolationForest

# ── 2. Initialise ─────────────────────────
model = IsolationForest(
    n_estimators=100,      # number of trees
    max_samples='auto',   # 256 or min(256,n)
    contamination=0.05,   # 5% expected anomalies
    random_state=42
)

# ── 3. Train ──────────────────────────────
model.fit(X_train)

# ── 4. Get values ─────────────────────────
# Labels: +1 = normal, -1 = anomaly
labels  = model.predict(X_test)

# Raw anomaly scores (lower = more anomalous)
scores  = model.decision_function(X_test)

# Normalised score (0 to 1, high = anomaly)
a_score = model.score_samples(X_test)

# ── 5. Filter anomalies ───────────────────
anomalies = X_test[labels == -1]
normals   = X_test[labels ==  1]

Type 03 · Density-Based Method
LOF — Local Outlier Factor
Breunig, Kriegel, Ng & Sander, 2000 · sklearn.neighbors.LocalOutlierFactor
LOF compares each point's local density to its neighbours' local densities — a point is anomalous if it exists in a significantly lower-density region than its surrounding neighbourhood, producing a dimensionless outlier factor score.
⚡ Core Insight: Global statistical methods fail when clusters have different densities — a point that's an outlier in a sparse cluster might look normal globally. LOF is local: it compares each point's density only to its immediate neighbours, making it robust to varying-density clusters.
Key Terms
k-Nearest Neighbours (k-NN): For each point, identify its k closest neighbours by distance. The parameter k (n_neighbors) controls how local "local" is. Larger k = smoother, less sensitive.
k-Distance d_k(x): The distance from x to its k-th nearest neighbour. Defines the boundary of x's local neighbourhood. Points within this radius are x's k-NN set N_k(x).
Reachability Distance rd_k(x, o): max(d_k(o), d(x, o)). The effective distance from x to neighbour o, floored by o's own k-distance. Smooths out local distance fluctuations.
Local Reachability Density lrd_k(x): Inverse of the average reachability distance of x to its k-NN. High lrd = dense neighbourhood. Low lrd = sparse neighbourhood.
LOF Score: LOF_k(x) = average ratio of neighbours' lrd to x's own lrd. LOF ≈ 1 → normal. LOF >> 1 → anomaly (x's neighbourhood is much less dense than its neighbours').
n_neighbors: The k parameter. Typical values: 10–50. Too small → noisy. Too large → misses local structure. Cross-validate or use domain knowledge to set.
Algorithm Steps
01
For each point x: find k nearest neighbours N_k(x)
Compute pairwise distances. Identify the k closest points. Record the k-distance d_k(x).
02
Compute reachability distance rd_k(x, o) for each neighbour
rd_k(x, o) = max(d_k(o), d(x, o)). This ensures nearby points don't get artificially tiny distances.
03
Compute lrd_k(x) — Local Reachability Density
lrd_k(x) = k / Σ rd_k(x, o) for all o in N_k(x). This is the reciprocal of the average reachability distance to neighbours.
04
Compute LOF_k(x) — Local Outlier Factor
LOF_k(x) = (Σ lrd_k(o) for o in N_k(x)) / (k · lrd_k(x)). Ratio of neighbours' density to x's density.
05
Threshold the LOF score
LOF ≈ 1 → inlier. LOF significantly > 1 (e.g., > 1.5 or > 2) → outlier. Threshold depends on dataset and domain.
Limitation: LOF is slow on large datasets (O(n²) distance computations). Also provides no natural threshold — LOF scores are relative, not absolute. Requires careful n_neighbors tuning.
Best for: Data with clusters of varying density. When you need to catch outliers that are "locally sparse" but globally ordinary. Spatial data, customer segments.
Code Syntax
lof.py
# ── 1. Import ─────────────────────────────
from sklearn.neighbors import LocalOutlierFactor

# ── 2. Initialise ─────────────────────────
lof = LocalOutlierFactor(
    n_neighbors=20,          # k parameter
    contamination=0.05,      # expected outlier %
    metric='euclidean'        # distance metric
)

# ── 3. Train + Predict (transductive) ────
# LOF is transductive by default in sklearn:
# fit_predict runs on the same data
labels  = lof.fit_predict(X)
# +1 = normal · -1 = anomaly

# ── 4. Get LOF scores ─────────────────────
# Negative LOF (sklearn convention: lower = more anomalous)
scores   = lof.negative_outlier_factor_
lof_vals = -scores   # flip sign → LOF > 1 = outlier

# ── For new data (inductive mode) ─────────
lof2 = LocalOutlierFactor(novelty=True, n_neighbors=20)
lof2.fit(X_train)
labels_new = lof2.predict(X_test)

Type 04 · Boundary-Based Method
One-Class SVM
Schölkopf et al., 2001 · sklearn.svm.OneClassSVM
One-Class SVM maps training data into a high-dimensional kernel space and finds the smallest hypersphere enclosing most of the data, treating points outside this learned boundary as anomalies — requiring only normal data for training.
⚡ Core Insight: Traditional SVMs separate two classes. One-Class SVM has only one class — normal data. It learns a tight boundary around the normal region in a kernelised feature space. Anything falling outside this boundary at test time is an anomaly. No anomaly examples are needed during training.
Key Terms
Kernel (RBF): A function that maps data into a high-dimensional space where a linear boundary can separate normal data from the origin. RBF (Radial Basis Function) kernel is the default and most common choice.
Hyperplane / Decision Boundary: In kernel space, a maximum-margin hyperplane separates the training data from the origin. In original input space, this corresponds to a curved, non-linear boundary around normal data.
ν (Nu) parameter: Controls the trade-off between model complexity and outlier tolerance. ν is an upper bound on the fraction of training errors (false positives) and a lower bound on support vectors. ν ∈ (0, 1].
Support Vectors: The training points that lie exactly on or near the decision boundary. They define the boundary shape. Points deep inside the boundary are ignored — only boundary points matter.
γ (Gamma): RBF kernel parameter controlling neighbourhood influence. High γ = tight, jagged boundary fitting closely to training points (risk: overfitting). Low γ = smooth, wide boundary.
Decision Function: Returns a signed distance from the boundary. Positive = inside boundary (normal). Negative = outside (anomaly). Magnitude = confidence of the classification.
Algorithm Steps
01
Map data to high-dimensional feature space via kernel
Using the RBF kernel φ(x), transform input data so that normal data forms a compact, separable cluster in the new space.
02
Find the maximum-margin hyperplane from the origin
Solve a quadratic programming problem to find the hyperplane w·φ(x) = ρ with maximum margin from the origin, enclosing most training points on the positive side.
03
Allow ν·n training points to be misclassified
The ν parameter permits a controlled fraction of training points to fall outside the boundary, making the model robust to noise in training data.
04
Identify support vectors defining the boundary
Only the points on or near the boundary (support vectors) define the model. Interior points are irrelevant once training is done.
05
At test time: evaluate sign of decision function
f(x) > 0 → inside boundary → normal (+1). f(x) < 0 → outside boundary → anomaly (−1).
Limitation: Very sensitive to kernel and hyperparameter choice (ν, γ). Slow on large datasets (O(n²) to O(n³) training). Requires clean training data — noisy training degrades the boundary significantly.
Best for: Medium-sized datasets, high-dimensional data, when only normal examples are available for training (novelty detection). Text, image, or network data where Isolation Forest underperforms.
Code Syntax
one_class_svm.py
# ── 1. Import ─────────────────────────────
from sklearn.svm import OneClassSVM
from sklearn.preprocessing import StandardScaler

# ── Scale first (SVM is distance-based) ──
scaler   = StandardScaler()
X_scaled = scaler.fit_transform(X_train)

# ── 2. Initialise ─────────────────────────
oc_svm = OneClassSVM(
    kernel='rbf',     # RBF kernel (default)
    nu=0.05,          # ~5% training outliers
    gamma='scale'     # auto-scales gamma
)

# ── 3. Train on NORMAL data only ──────────
oc_svm.fit(X_scaled)

# ── 4. Get values ─────────────────────────
# Labels: +1 = normal · -1 = anomaly
labels   = oc_svm.predict(X_test_scaled)

# Signed distance from boundary
scores   = oc_svm.decision_function(X_test_scaled)

# ── 5. Filter anomalies ───────────────────
anomalies = X_test[labels == -1]
normals   = X_test[labels ==  1]

Decision Guide — Which Method to Use?
Data is roughly Gaussian?
Start with Z-Score. Fast, transparent, no training needed.
Large, messy, high-dim dataset?
Use Isolation Forest. Robust, fast, minimal tuning.
Clusters with different densities?
Use LOF. Only local comparison matters, not global.
Only normal examples available?
Use One-Class SVM. Learns tight boundary from clean data.
