Topic 01
Clustering
One-word definition Grouping
Clustering is an unsupervised machine learning technique that partitions unlabelled data into meaningful groups based on intrinsic similarity, enabling pattern discovery without prior class definitions or target variables.
Types of Clustering Algorithms
Centroid-Based
Groups data around central points (centroids). Partition-based, requires K upfront.
→ K-Means, K-Medoids
Connectivity-Based (Hierarchical)
Builds a tree (dendrogram) by iteratively merging or splitting clusters.
→ Agglomerative, Divisive
Density-Based
Defines clusters as dense regions separated by sparse space. Handles noise natively.
→ DBSCAN, HDBSCAN, OPTICS
Distribution-Based
Assumes data fits a statistical distribution (e.g. Gaussian). Soft cluster membership.
→ Gaussian Mixture Models (GMM)
Centroid-Based · Algorithm 01
K-Means
centroid-based
How it works — Steps
01
Choose K
Decide how many clusters to form. Use the Elbow Method: plot WCSS vs K, pick the bend.
02
Place K centroids randomly
Scatter K random seed points in the feature space.
03
Assign each point to nearest centroid
Measure Euclidean distance from each point to every centroid. Assign to the closest.
04
Recompute centroids
New centroid = mean of all points currently in that cluster.
05
Repeat until convergence
Iterate steps 3–4 until centroids no longer move (or max iterations reached).
WCSS / Inertia = sum of squared distances of each point to its assigned centroid. Lower = tighter clusters.
Limitation: Assumes spherical clusters · Sensitive to outliers · Must pre-define K.
Code Syntax
kmeans.py 
# ── 1. Import ────────────────────────
from sklearn.cluster import KMeans

# ── 2. Initialise ────────────────────
model = KMeans(
    n_clusters=3,
    init='k-means++',   # smart seeding
    random_state=42
)

# ── 3. Train ─────────────────────────
model.fit(X)

# ── 4. Get values ────────────────────
labels     = model.labels_           # cluster per point
centroids  = model.cluster_centers_  # centroid coords
inertia    = model.inertia_          # WCSS score

# ── 5. Predict new points ────────────
pred = model.predict(X_new)
Connectivity-Based · Algorithm 02
Hierarchical Clustering
dendrogram-based
↑ BOTTOM-UP
Agglomerative
Starts with each point as its own cluster. Merges up. Most common approach.
↓ TOP-DOWN
Divisive
Starts with all points in one cluster. Splits down. Less common but more precise.
↑
Agglomerative BOTTOM-UP · merge
Steps
01
Every point is its own cluster
Start with N clusters (one per data point).
02
Compute pairwise distances
Calculate distance between every cluster pair using the chosen linkage method.
03
Merge the two closest clusters
Combine the most similar pair into one new cluster. Record the merge height.
04
Repeat until one cluster remains
Each merge builds the dendrogram tree from the leaves up to the root.
05
Cut the dendrogram
Draw a horizontal line at a chosen height. Lines crossed = number of clusters.
Linkage: ward (best default — minimises variance) · complete · average · single
Code Syntax
agglomerative.py 
# ── 1. Import ─────────────────────
from sklearn.cluster import AgglomerativeClustering
import scipy.cluster.hierarchy as sch

# ── Visualise dendrogram first ────
sch.dendrogram(sch.linkage(X, method='ward'))

# ── 2. Initialise ─────────────────
model = AgglomerativeClustering(
    n_clusters=3,
    linkage='ward'
)

# ── 3. Train ──────────────────────
model.fit(X)

# ── 4. Get values ─────────────────
labels = model.labels_

# ⚠ No .predict() — re-fit for new data
↓
Divisive TOP-DOWN · split
Steps
01
All points form one giant cluster
Start with the entire dataset as a single cluster — the root of the tree.
02
Find the most heterogeneous cluster
Identify the cluster with the highest internal dissimilarity (most spread out).
03
Split it into two sub-clusters
Use a flat clustering method (e.g. K-Means with K=2) to divide it optimally.
04
Repeat on each sub-cluster
Continue splitting the most heterogeneous cluster at each step, building the tree downward.
05
Stop at K clusters or leaf nodes
Stop when each point is its own cluster, or cut the tree at the desired level of K.
Key difference from Agglomerative: Decisions made early (at the top) are final — there is no merging back. This makes divisive more computationally expensive but more globally aware.
Sklearn note: No native divisive implementation. Use sklearn + scipy manually, or the DIANA algorithm via the pyclust library.
Code Syntax (manual implementation)
divisive.py 
# ── 1. Import ─────────────────────
from sklearn.cluster import KMeans
import numpy as np

# ── 2. Recursive split function ───
def divisive_split(X, min_size=2):
    if len(X) <= min_size:
        return [X]
    # Split current cluster into 2
    km = KMeans(n_clusters=2, random_state=42)
    labels = km.fit_predict(X)
    left  = X[labels == 0]
    right = X[labels == 1]
    # Recurse on each sub-cluster
    return divisive_split(left) + \
           divisive_split(right)

# ── 3. Run ────────────────────────
clusters = divisive_split(X)

# ── 4. Get values ─────────────────
# clusters = list of sub-arrays,
# each sub-array = one final cluster
Agglomerative is preferred when:
You don't know K. You want to explore all granularities. Dataset is small-to-medium.
Divisive is preferred when:
You have a clear top-level split in mind. You want a more globally consistent tree structure.
Density-Based · Algorithm 03
DBSCAN
Density-Based Spatial Clustering of Applications with Noise
density-based
eps
Radius of the neighbourhood around a point
min_samples
Min. points inside eps to be a core point
label = -1
Points that don't belong to any cluster (outliers)
How it works — Steps
01
Pick any unvisited point
Draw a circle of radius eps around it.
02
Count neighbours inside eps
If count ≥ min_samples → core point. Else → border or noise.
03
Expand the cluster
Add all density-connected points. If they are also core points, expand further.
04
Move to next unvisited point
Repeat steps 1–3 until all points are visited.
05
Label unreached points as noise
Points never added to any cluster get label = -1.
Superpower: Finds arbitrary-shaped clusters. Auto-detects outliers. No K needed.
Limitation: Struggles with varying density. Very sensitive to eps and min_samples values.
Code Syntax
dbscan.py 
# ── 1. Import ────────────────────────
from sklearn.cluster import DBSCAN

# ── 2. Initialise ────────────────────
model = DBSCAN(
    eps=0.5,        # neighbourhood radius
    min_samples=5   # core point threshold
)

# ── 3. Train ─────────────────────────
model.fit(X)

# ── 4. Get values ────────────────────
labels       = model.labels_             # -1 = outlier
core_idx     = model.core_sample_indices_

# Count real clusters (exclude noise)
n_clusters = len(set(labels)) - (
    1 if -1 in labels else 0
)
Clustering — Quick Comparison
Algorithm Needs K? Handles Outliers? Cluster Shape Best For
K-Means ✕ Yes (required) ✕ No Spherical Large, well-separated, similarly-sized blobs
Hierarchical ~ Post-hoc cut Partially Any (ward → compact) Small datasets, exploring hierarchy of clusters
DBSCAN ✓ No ✓ Yes (native) Arbitrary Irregular shapes, anomaly detection
Topic 02
Dimensionality Reduction
One-word definition Compression
Dimensionality reduction transforms high-dimensional data into a lower-dimensional representation, preserving the most statistically significant structure while discarding redundant features, noise, and collinear information that impede model performance.
Why reduce dimensions?
Speed
Fewer features → faster training & inference
Visualise
Reduce to 2D/3D for human-readable plots
Noise
Remove low-variance, redundant features
Overfitting
Simpler representation → better generalisation
Types of Dimensionality Reduction Algorithms
Linear Methods
Find linear combinations of features that preserve structure (variance or class separability).
→ PCA, LDA, Factor Analysis, ICA
Non-Linear (Manifold) Methods
Capture complex, curved structure in data that linear methods miss.
→ t-SNE, UMAP, Kernel PCA, Isomap
Matrix Factorisation
Decompose the data matrix into smaller factor matrices.
→ SVD, NMF, Truncated SVD
Autoencoder (Deep Learning)
Neural networks that learn compressed representations in a bottleneck layer.
→ Vanilla, Variational (VAE), Sparse AE
Linear · Unsupervised · Algorithm 01
PCA
Principal Component Analysis
unsupervised
Core Intuition: PCA finds the directions in which your data varies the most, ranks them by importance, and projects your data onto the top-K directions — discarding the rest. It doesn't use labels; it purely follows variance.
Step-by-Step — PCA Internals
01
Standardise the data
Scale each feature to mean=0, std=1. PCA is variance-based, so large-scale features dominate without scaling.
02
Compute the Covariance Matrix
An n×n matrix showing how every pair of features co-varies. Captures linear relationships between features.
03
Compute Eigenvalues & Eigenvectors
Eigenvectors = directions of max variance (the axes). Eigenvalues = magnitude of variance in each direction.
04
Sort by Eigenvalue (descending)
PC1 = most variance, PC2 = second most, … Each PC is orthogonal (perpendicular) to all others.
05
Select top K components
Choose K so that cumulative explained variance ≥ 95%. Use Scree Plot to visualise the elbow.
06
Project data onto new axes
Multiply original data by the K eigenvectors. Result: a dataset with K columns instead of n.
Key Terms
Principal Component (PC)
A new axis = a linear combination of all original features. PCs always point in the direction of maximum remaining variance and are perpendicular to each other.
Explained Variance Ratio
What fraction of total variance does each PC capture? Sum them cumulatively to see how much information you retain.
Loadings
Values in each eigenvector. High loading = that original feature contributes strongly to that PC.
Loss of interpretability
PC1 is NOT a single original feature — it's a weighted mix of all features. You can no longer say "this is the age column."
Code Syntax
pca.py 
# ── 1. Import ────────────────────────
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# ── Standardise first (mandatory) ────
scaler   = StandardScaler()
X_scaled = scaler.fit_transform(X)

# ── 2. Initialise ────────────────────
pca = PCA(n_components=2)
# Or keep 95% variance: PCA(n_components=0.95)

# ── 3. Train ─────────────────────────
pca.fit(X_scaled)

# ── 4. Get values ────────────────────
X_pca      = pca.transform(X_scaled)
var_ratio  = pca.explained_variance_ratio_
cum_var    = var_ratio.cumsum()
components = pca.components_  # loadings

# ── Shortcut ─────────────────────────
X_pca = pca.fit_transform(X_scaled)
Linear · Supervised · Algorithm 02
LDA
Linear Discriminant Analysis
supervised
Core Intuition: LDA finds axes that maximise the separation between classes while minimising the spread within each class. Unlike PCA which ignores labels, LDA uses class labels to find the most discriminative direction.
PCA asks → "Where is the variance?"
Projects onto axes of maximum total variance. Label-blind. Goal: preserve as much data spread as possible.
↑ explained variance
LDA asks → "Where is the separation?"
Projects onto axes of maximum between-class separation. Uses labels. Goal: make classes as distinct as possible.
↑ SB / ↓ SW
Step-by-Step — LDA Internals
01
Compute class means
Calculate a mean vector (centroid) for each class label separately.
02
Compute Within-Class Scatter (SW)
Sum of covariance matrices per class. Measures how tightly grouped points are inside each class. We want SW small.
03
Compute Between-Class Scatter (SB)
Measures how spread class centroids are from the overall mean. We want SB large.
04
Solve SW⁻¹ · SB (Generalised Eigenvalue problem)
Find eigenvectors that maximise the ratio SB/SW — these are the linear discriminants.
05
Select top K discriminants
Maximum K = (number of classes − 1). With 3 classes, you get at most 2 LDA components.
06
Project data onto discriminant axes
New dataset has K columns. Classes should now be visually and geometrically separated.
Key Terms
Linear Discriminant (LD)
A new axis that maximally separates class means while keeping within-class spread compact.
Max components = C − 1
LDA is fundamentally limited by number of classes. Binary (2-class) problem → only 1 LD axis possible.
SW & SB matrices
SW = within-class scatter (want small). SB = between-class scatter (want large). LDA maximises SB/SW.
Assumptions
Assumes normally distributed classes with equal covariance per class. Violations reduce effectiveness.
Code Syntax
lda.py 
# ── 1. Import ────────────────────────
from sklearn.discriminant_analysis import \
    LinearDiscriminantAnalysis

# ── 2. Initialise ────────────────────
lda = LinearDiscriminantAnalysis(n_components=2)
# Max: n_components = n_classes - 1

# ── 3. Train (y required!) ───────────
lda.fit(X, y)   # ← y is REQUIRED, unlike PCA

# ── 4. Get values ────────────────────
X_lda     = lda.transform(X)
var_ratio = lda.explained_variance_ratio_
scalings  = lda.scalings_  # discriminant vectors

# ── Shortcut ─────────────────────────
X_lda = lda.fit_transform(X, y)

# ── LDA can also classify directly ───
y_pred = lda.predict(X_test)

Decision Guide — Which to use?
Have labels + classifying?
Use LDA to reduce first, then pass to your classifier.
No labels or just exploring?
Use PCA for compression, noise removal, or 2D visualisation.
Finding groups in data?
Irregular shapes → DBSCAN. Need K? → K-Means.
