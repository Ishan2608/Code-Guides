Section 01

Build the Mental Model First

Before touching any library, understand the three-layer structure that every Python visualization sits inside. Get this wrong and even simple charts become confusing.

Think of a chart the same way you'd think about a physical painting. There is a canvas (Figure), a painted panel on the canvas (Axes), and the actual drawing inside that panel. You almost never draw directly on the canvas — you get a panel first, then you work inside it.

Figure   — the entire window / image file
↓ contains one or more
Axes   — a single chart panel with its own X and Y
↓ contains
Lines
Bars
Fills
Scatter
Ticks / Labels
Key Insight
When you write fig, ax = plt.subplots(), you are creating one Figure and one Axes simultaneously, getting references to both so you can control them independently. The ax object is where 95% of your work happens.

Two Ways to Write Matplotlib

You will see both styles in the wild. Always prefer the object-oriented (OOP) style — it is explicit and scales to multi-panel charts.

Avoid — pyplot style
# "Stateful" — matplotlib tracks a hidden current figure
plt.plot(x, y)
plt.xlabel("Date")
plt.title("My Chart")
plt.show()
Simple, but breaks down with multiple panels
Use This — OOP style
# Explicit — you control every object directly
fig, ax = plt.subplots()
ax.plot(x, y)
ax.set_xlabel("Date")
ax.set_title("My Chart")
plt.show()
Works whether you have 1 panel or 10

The rcParams — Global Style Configuration

Instead of setting colors and fonts on every single chart, plt.rcParams is a global dictionary that sets defaults for your entire session. Think of it as a CSS stylesheet for all your matplotlib plots. In the notebook, this is done once at the top to establish the dark terminal theme. 

# Set once. Applies to ALL subsequent charts in the session.
plt.rcParams.update({
    "figure.facecolor" : "#0e0f14",  # background of the Figure
    "axes.facecolor"   : "#15161d",  # background of each Axes panel
    "axes.grid"        : True,
    "grid.color"       : "#1e1f2e",
    "text.color"       : "#d4d5e8",
    "figure.dpi"       : 150,       # higher = crisper image output
    "font.family"      : "monospace",
})
Section 02

Matplotlib Deep Dive

The building blocks that appear in almost every single chart. Master these and every plot becomes a remix of the same parts.

Core Method
ax.fill_between() — Shaded Regions

Simple version: fill the area between a line and a flat baseline (e.g. the bottom of the chart). The power version: fill between two lines, but only where a condition is true. This creates the bull/bear zone coloring in the notebook. 

# Basic: fill area under a price line
ax.fill_between(
    df.index,           # X values (dates)
    df["Close"].min(), # bottom boundary (a flat number)
    df["Close"],       # top boundary (the line itself)
    alpha=0.18,         # transparency: 0.0 = invisible, 1.0 = solid
    color="#7c6af7"
)

# Conditional: fill between MA20 and MA50 only when MA20 is higher
ax.fill_between(
    df.index, df["MA20"], df["MA50"],
    where=df["MA20"] > df["MA50"],  # boolean Series — the key parameter
    alpha=0.08,
    color="#26c980",
    label="Bull Zone"
)
Plain English
"Paint the area between these two things, but only in the spots where this condition is true." The where parameter accepts any boolean array with the same length as your data.
Core Method
ax.axhline() / ax.axvline() — Reference Lines

Draw a horizontal or vertical line that spans the full width/height of the chart. Used extensively for threshold lines (RSI 70/30, zero line on MACD) and median reference lines. 

# RSI overbought threshold
ax.axhline(70, color="#f76a6a", lw=0.8, linestyle="--", alpha=0.7)

# Vertical line at a specific date event
ax.axvline("2024-04-09", color="#f7b955", lw=1.0, linestyle=":")
Core Method
ax.annotate() / ax.text() — Labels on the Chart

ax.text(x, y, "string") places text at absolute chart coordinates. ax.annotate() is more powerful — it places text relative to a point and can optionally draw an arrow pointing at that point. 

# Simple text label at a data point
ax.text(
    years[i], value + 0.5,  # x, y position
    f"{value:.1f}%",          # the string to show
    ha="center",             # horizontal alignment
    va="bottom",             # vertical alignment
    fontsize=7,
    color="#d4d5e8"
)

# Annotate with offset — useful when the point is crowded
ax.annotate(
    f"{value:.0f}",
    xy=(date, value),              # the point you're labeling
    xytext=(5, 0),               # offset in points from xy
    textcoords="offset points", # unit of xytext offset
    color="#5de8c1",
    fontsize=8,
    va="center"
)
Pattern
Annotating Bar Charts — The Standard Pattern

Placing value labels on top of each bar is a pattern you will use constantly. You iterate over the bars object returned by ax.bar() to get the position and height of each bar. 

bars = ax.bar(years, values, color=colors)

for bar, val in zip(bars, values):
    ax.text(
        bar.get_x() + bar.get_width() / 2,  # center of bar on X
        bar.get_height() * 1.01,            # just above the bar top
        f"{val:.1f}%",
        ha="center",
        fontsize=7.5
    )
Pattern
Grouped Bar Charts — The offset trick

To place multiple bar groups side-by-side (like Revenue, Gross Profit, Net Income per year), you shift each group's X position by a fraction of the bar width. NumPy's arange() gives you integer positions, then you add or subtract offsets. 

import numpy as np
x = np.arange(len(years))  # [0, 1, 2, 3] — integer positions
width = 0.25               # width of each individual bar

ax.bar(x - width,  revenue,  width, label="Revenue",     color=C_LINE)
ax.bar(x,          gross_p,  width, label="Gross Profit", color=C_MA50)
ax.bar(x + width,  net_inc,  width, label="Net Income",   color=C_UP)

ax.set_xticks(x)            # place ticks at integer positions
ax.set_xticklabels(years)   # replace integers with year strings
Mental Model
The numbers 0, 1, 2, 3 are just positions on a ruler. You then manually glue year labels onto those positions with set_xticklabels(). This lets you precisely control spacing between groups.
Styling
Spines — The Chart Border Lines

Every chart has four border lines — top, right, left, bottom. These are called "spines". Most modern chart styles remove all of them except the bottom, giving a cleaner open look. The notebook defines a helper function for this. 

def clean_spines(ax, keep=None):
    keep = keep or []
    for spine in ["top", "right", "left", "bottom"]:
        ax.spines[spine].set_visible(spine in keep)

# Usage examples
clean_spines(ax, keep=["bottom"])         # most charts in the notebook
clean_spines(ax, keep=["bottom", "left"]) # scatter plots
clean_spines(ax)                            # remove all spines

The Date Axis — Locators & Formatters

This is the part that feels most magical and confusing at first. Demystified below.

When your X axis contains dates, matplotlib needs to answer two questions: (1) where to put the tick marks (which dates get a line?) and (2) how to label them (what text to show?). These are two separate jobs done by two separate objects.

Locator
answers WHERE
+
Formatter
answers WHAT TEXT
→
Readable X axis
Jan '24, Mar '24...
Locators — WHERE to put ticks
import matplotlib.dates as mdates

# Tick every 2 months
ax.xaxis.set_major_locator(
    mdates.MonthLocator(interval=2)
)

# Tick every year
ax.xaxis.set_major_locator(
    mdates.YearLocator()
)

# Tick every week
ax.xaxis.set_major_locator(
    mdates.WeekdayLocator()
)
Formatters — WHAT TEXT to show
# Format codes are standard Python strftime
# %b = abbreviated month (Jan, Feb...)
# %Y = 4-digit year    %y = 2-digit year

# Shows: "Jan '24"
ax.xaxis.set_major_formatter(
    mdates.DateFormatter("%b '%y")
)

# Shows: "Jan, 24" (with comma)
ax.xaxis.set_major_formatter(
    mdates.DateFormatter("%b, %y")
)

The Y-Axis — FuncFormatter for Custom Labels

For the Y-axis (price, volume, etc.), you use a different module: matplotlib.ticker. The most flexible tool is FuncFormatter — you give it a Python function, and it calls that function on every tick value to produce the label string. 

import matplotlib.ticker as mticker

# FuncFormatter takes a function(value, position) -> string
# The 'position' argument is tick index — you usually ignore it (use _)

# Show prices with ₹ and comma formatting: 3500.0 → "₹3,500"
ax.yaxis.set_major_formatter(
    mticker.FuncFormatter(lambda x, _: f"₹{x:,.0f}")
)

# Show volume in millions: 5000000 → "5M"
ax.yaxis.set_major_formatter(
    mticker.FuncFormatter(lambda x, _: f"{x:.0f}M")
)

# Show percentages: 0.245 → "24.5%"
ax.yaxis.set_major_formatter(
    mticker.FuncFormatter(lambda x, _: f"{x*100:.1f}%")
)
Plain English
"For every tick mark on the Y axis, call my function with the raw number, and use whatever string my function returns as the label." This is why you can show "₹3,500" instead of "3500.0" — you're literally formatting the number yourself inside that lambda.

Hiding Tick Labels Between Linked Panels

When you have a price chart above a volume chart, you share the X axis (sharex=ax_price). This keeps both panels in sync when you zoom. You then hide the date labels on the top panel because the bottom panel already shows them — avoiding duplication. 

# Hide x-tick labels on the price panel (top) — bottom panel shows them
plt.setp(ax_price.get_xticklabels(), visible=False)
Section 04

Multi-Panel Layouts — subplots vs GridSpec

Two tools for placing multiple charts inside one Figure. Choose based on whether you need equal or unequal panel sizes.

plt.subplots() — Equal panels

When all panels should be the same size. Returns a 2D array of Axes objects.

# 1 row, 2 columns → 2 charts side by side
fig, (ax_left, ax_right) = plt.subplots(
    1, 2,
    figsize=(16, 8)
)

# 2 rows, 3 columns → 6 charts in a grid
fig, axes = plt.subplots(2, 3, figsize=(16, 8))
axes[0, 0]  # top-left
axes[1, 2]  # bottom-right
GridSpec — Unequal panels

When panels should have different sizes — price chart 3x taller than volume, for example.

from matplotlib.gridspec import GridSpec

# Think of it as a 4-row grid
fig = plt.figure(figsize=(14, 7))
gs = GridSpec(4, 1, figure=fig, hspace=0.05)

# Price takes rows 0-2, volume takes row 3
ax_price = fig.add_subplot(gs[:3, 0])
ax_vol   = fig.add_subplot(gs[3, 0],
                            sharex=ax_price)
GridSpec Mental Model
GridSpec is like defining a grid of rows and columns — then you say "give me a panel that spans rows 0 to 3 in column 0." The slice notation gs[:3, 0] is exactly Python list slicing — take rows 0, 1, 2 and column 0. This lets one panel occupy multiple grid cells, making it bigger than others.

The hspace / wspace Parameters

hspace controls vertical space between rows. wspace controls horizontal space between columns. In the notebook, hspace=0.05 makes panels nearly touching — creating the "unified dashboard" look. plt.tight_layout() then automatically adjusts all margins to prevent label overlap.

The Technical Dashboard Pattern

The 4-panel technical analysis chart (Price/Bollinger, RSI, MACD, ATR) uses a 10-row GridSpec so each indicator gets a proportional number of rows: 

# 10-row grid: price gets 4 rows, each indicator gets 2 rows
gs = GridSpec(10, 1, figure=fig, hspace=0.06)

ax1 = fig.add_subplot(gs[:4])          # rows 0–3  (40%)
ax2 = fig.add_subplot(gs[4:6], sharex=ax1)  # rows 4–5  (20%)
ax3 = fig.add_subplot(gs[6:8], sharex=ax1)  # rows 6–7  (20%)
ax4 = fig.add_subplot(gs[8:],  sharex=ax1)  # rows 8–9  (20%)

# All panels share the same X axis → synced zooming
Section 05

mplfinance — Candlestick Charts

A specialized library built on top of matplotlib specifically for financial OHLCV charts. It handles candlestick rendering so you don't have to draw each candle manually.

What is a Candlestick?
Each "candle" represents one time period. The body shows Open and Close. The thin lines (wicks/shadows) extend to the High and Low. Green body = price went up that period. Red body = price went down. This packs four values into one visual element.

How mplfinance Works

Unlike raw matplotlib where you manually call ax.plot() for everything, mplfinance has a single main function mpf.plot() that does all the heavy lifting. You pass it your DataFrame and a style object, and it generates the chart.

Step 1 — Create the color scheme
# marketcolors defines what each part of a candle looks like
mc = mpf.make_marketcolors(
    up="#26c980",    # color of green (up) candles
    down="#f76a6a",  # color of red (down) candles
    wick={"up": "#26c980", "down": "#f76a6a"},
    volume={"up": "#26c980", "down": "#f76a6a"},
    edge="none"       # no border around candle body
)
Step 2 — Create the full style
# make_mpf_style bundles marketcolors + background colors + fonts
custom_style = mpf.make_mpf_style(
    base_mpl_style="dark_background", # start from a built-in theme
    marketcolors=mc,                  # apply our candle colors
    facecolor="#0e0f14",              # axes background
    figcolor="#0e0f14",               # figure background
    gridcolor="#1e1f2e",
    gridstyle="--"
)
Step 3 — Plot
# df_3mo must have columns: Open, High, Low, Close, Volume
# and a DatetimeIndex — exactly what yfinance gives you
fig, axes = mpf.plot(
    df_3mo,
    type="candle",       # "candle", "ohlc", "line", "renko"
    style=custom_style,
    mav=(20, 50),         # overlay moving averages
    volume=True,          # add volume panel automatically
    figsize=(14, 7),
    title="TCS.NS",
    tight_layout=True,
    returnfig=True,       # return (fig, axes) so you can save it
    mavcolors=["#f7b955", "#5de8c1"]
)
Why returnfig=True?
By default mpf.plot() shows the chart and returns nothing. Setting returnfig=True makes it return the (fig, axes) tuple so you can then call fig.savefig() to save the chart to disk.
Section 06

Plotly — Interactive Charts

Completely different paradigm from matplotlib. Instead of drawing on a canvas, you build a data structure (a Figure object) and Plotly renders it as an interactive HTML widget.

Dimension Matplotlib Plotly
Output Static PNG / inline image Interactive HTML (hover, zoom, pan, toggle)
Mental model Canvas + Axes + drawing commands Figure object + list of traces (data layers)
Configuration Method calls on Axes Dictionary/keyword arguments in update_layout()
Use case Publication, reports, tight control Dashboards, exploration, Jupyter notebooks

The Core Mental Model — Figures and Traces

A Plotly figure is a container. You add "traces" to it — each trace is one data series (one line, one set of candles, etc.). The figure renders all traces layered on top of each other. Think of it like Photoshop layers.

Pattern — Interactive Candlestick with Overlays
import plotly.graph_objects as go

# Step 1: Create empty figure container
fig = go.Figure()

# Step 2: Add traces (data layers)

# Trace 1 — Candlestick
fig.add_trace(go.Candlestick(
    x=df.index,
    open=df["Open"],
    high=df["High"],
    low=df["Low"],
    close=df["Close"],
    name="OHLC",
    increasing_line_color="#26c980",
    decreasing_line_color="#f76a6a"
))

# Trace 2 — MA20 line overlaid on same axes
fig.add_trace(go.Scatter(
    x=df.index,
    y=df["Close"].rolling(20).mean(),
    line=dict(color="#f7b955", width=1),
    name="MA 20"
))

# Step 3: Configure global layout
fig.update_layout(
    title="TCS.NS — Interactive Candlestick",
    template="plotly_dark",              # built-in dark theme
    xaxis_rangeslider_visible=False,   # hide the range slider below
    height=500,
    paper_bgcolor="#0e0f14",
    plot_bgcolor="#15161d",
    font=dict(family="monospace", size=10)
)

fig.show()   # renders as interactive HTML in Jupyter
Key Difference from Matplotlib
Notice there is no ax object. Instead of calling methods on an axes, you pass all configuration as keyword arguments to update_layout(). The "trace" concept is equivalent to calling ax.plot() or ax.bar() — each one adds another data layer.

go.Scatter vs go.Candlestick vs go.Bar

These are the three main trace types you'll use with financial data. go.Scatter covers both lines and scatter points (use mode="lines" or mode="markers"). go.Candlestick requires OHLC columns. go.Bar is for bar charts.

Seaborn — Statistical Charts

Built on matplotlib, seaborn adds high-level functions for statistical visualizations. You pass a DataFrame and column names — it handles the math.

The notebook uses seaborn for exactly one chart: the correlation heatmap. This is where seaborn shines — charts that would require significant math + drawing code in raw matplotlib are a single function call in seaborn.

Correlation Heatmap
import seaborn as sns

# corr is a DataFrame of correlation values (-1.0 to 1.0)
corr = returns.corr()

# diverging_palette creates a colormap that goes
# blue (negative) → white (zero) → green (positive)
cmap = sns.diverging_palette(250, 130, s=80, as_cmap=True)

sns.heatmap(
    corr,
    annot=True,            # print the number inside each cell
    fmt=".2f",              # format those numbers to 2 decimal places
    cmap=cmap,
    center=0.5,            # value that maps to the center of the colormap
    linewidths=1,          # width of grid lines between cells
    linecolor="#080a0f",   # color of those grid lines
    annot_kws={"size": 9},  # font size of the numbers inside cells
    ax=ax,                  # draw into an existing matplotlib Axes
    cbar_kws={"shrink": 0.8} # make the colorbar legend 80% height
)
Important
Notice ax=ax at the end. Seaborn functions accept an ax parameter, meaning they draw into an existing matplotlib Axes. This is how you embed seaborn charts inside a larger matplotlib figure with GridSpec or subplots.

What does a diverging colormap mean?

A diverging colormap has a meaningful center. For correlation: 0.0 means no relationship (neutral color), +1.0 means perfectly correlated (one extreme color), -1.0 means perfectly inversely correlated (opposite extreme color). The center parameter tells seaborn which value should map to the neutral middle of the colormap.

Section 08

Cross-Cutting Patterns

Things that appear in almost every chart regardless of library. Internalize these and you'll recognize the structure of any chart you encounter.

Pattern 1 — zorder: Who Draws on Top?

When multiple elements overlap, zorder controls the drawing order. Higher number = drawn later = appears on top. Think of it like Z-index in CSS. 

# Grid lines drawn first (low zorder), price line on top of them
ax.plot(df.index, df["Close"], zorder=4)
ax.fill_between(..., zorder=2)
# ax.grid() is drawn at zorder 0 by default

Pattern 2 — The alpha Parameter

alpha sets transparency on any element. 0.0 = invisible, 1.0 = fully opaque. Filled regions almost always use a low alpha (0.06 – 0.20) so they don't overpower the lines drawn on top.

Pattern 3 — lw, ls, marker

# lw / linewidth — thickness of the line
ax.plot(x, y, lw=1.5)         # standard line
ax.plot(x, y, lw=0.8)         # thin/secondary line

# linestyle — dashed, dotted, dash-dot
ax.plot(x, y, linestyle="--")  # dashed   → MA lines
ax.plot(x, y, linestyle=":")   # dotted   → reference lines
ax.plot(x, y, linestyle="-.")  # dash-dot → MA200

# marker — a symbol drawn at each data point
ax.plot(years, values, marker="o", markersize=4)  # circle
ax.plot(years, values, marker="s")                 # square
ax.plot(years, values, marker="^")                 # triangle

Pattern 4 — fig.suptitle() vs ax.set_title()

ax.set_title() puts a title inside a single panel. fig.suptitle() puts a title above the entire figure — used when you have multiple panels and want one overall heading. 

fig.suptitle("TCS — Key Financial Ratios Dashboard", fontsize=13, y=1.01)
# y=1.01 nudges it slightly above the top panel to avoid overlap

Pattern 5 — Normalizing for Comparison

When comparing multiple stocks with different price levels (TCS at ₹3500, Wipro at ₹280), you can't plot raw prices on the same chart meaningfully. You normalize them to a common base of 100 on day 1, so you're comparing percentage returns, not absolute prices. 

# Each stock starts at 100. After 1 year at +25%, it shows 125.
norm = (prices / prices.iloc[0]) * 100

# Then plot all normalized series on the same ax
for col in norm.columns:
    ax.plot(norm.index, norm[col], label=col)

Pattern 6 — Scatter Plot Quadrant Labels

The Risk vs Return scatter plot divides the chart into four quadrants using two reference lines, then uses ax.transAxes to place text at fractional positions relative to the chart size (0.0 = left/bottom, 1.0 = right/top). 

# ax.transAxes means (0.02, 0.97) = 2% from left, 97% from bottom
# This is independent of the actual data values on X and Y
ax.text(0.02, 0.97, "Low Risk\nHigh Return",
        transform=ax.transAxes,  # key: use axes coords not data coords
        color="#26c980", fontsize=8, va="top")

Pattern 7 — Saving Charts

# Always save BEFORE plt.show()
# plt.show() resets the figure; saving after gives you a blank file
plt.tight_layout()
fig.savefig("chart.png", bbox_inches="tight", dpi=150)
# bbox_inches="tight" prevents labels from being cropped at the edges
plt.show()

Section 09

How to Read Any Unfamiliar Chart Code

A repeatable 5-step approach whenever you encounter visualization code you haven't seen before.

Step 1
Identify the library

Is it plt. / ax.? That's matplotlib. go. or fig.add_trace()? Plotly. mpf.? mplfinance. sns.? Seaborn. Each has its own mental model.

Step 2
Find the figure and axes creation

Look for plt.subplots(), plt.figure(), go.Figure(), or GridSpec(). This tells you how many panels exist and their relative sizes.

Step 3
Find the data calls

What is being drawn? ax.plot() = line. ax.bar() = bars. ax.scatter() = dots. ax.fill_between() = shaded region. Each one adds one visual layer.

Step 4
Find the axis configuration

Look for set_major_formatter, set_major_locator, set_xlabel, set_title, set_ylabel. These are cosmetic — they label the chart and format the tick text.

Step 5
Read the legend and style calls

ax.legend(), clean_spines(), plt.tight_layout(), fig.savefig() — these are final polish steps. They don't change the data being shown, just how it looks and where it's saved.

The Most Important Realization
Every chart in the notebook — even the complex 4-panel technical dashboard — is built from the same ~10 primitives: plot(), bar(), scatter(), fill_between(), axhline(), text(), annotate(), formatters, locators, and spine/grid styling. Once you understand these building blocks, you can reverse-engineer any chart.
