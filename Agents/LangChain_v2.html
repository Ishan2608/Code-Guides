Chat Models

Two ways to instantiate a model. init_chat_model is preferred for flexibility; the provider class gives full parameter control.

2.1 — Using init_chat_model

Takes a "provider:model-name" string. Easiest to swap providers.

from langchain.chat_models import init_chat_model

# Format: "provider:model-name"
llm = init_chat_model(f"google_genai:{GEMINI_MODEL}", api_key=GEMINI_API_KEY)

response = llm.invoke("Explain the subconscious mind in 100 words.")

print(response.text)          # shorthand for text content
print(response.content)       # same (str for most providers)
print(response.usage_metadata)  # token counts
Provider StringPackage Required
"google_genai:gemini-2.0-flash"langchain-google-genai
"openai:gpt-4.1"langchain-openai
"anthropic:claude-sonnet-4-6"langchain-anthropic
"groq:llama-3.1-8b-instant"langchain-groq

2.2 — Using a Provider Class

Direct instantiation — full access to model-specific parameters.

from langchain_google_genai import ChatGoogleGenerativeAI

llm = ChatGoogleGenerativeAI(
    model=GEMINI_MODEL,
    api_key=GEMINI_API_KEY,
    temperature=0.7,          # 0.0 = deterministic, 1.0 = creative
    max_tokens=512,
)

response = llm.invoke("Write a formal 100-word description of artificial intelligence.")
print(response.content)
response.content is always a string for text models. Use response.text as a convenience alias. Both give you the model's reply.

Message Templates

LangChain models accept a list of messages, not just a plain string. Use message objects for clarity.

from langchain_core.messages import SystemMessage, HumanMessage, AIMessage
ClassRoleEquivalent Dict
SystemMessageInstructions / persona for the model{"role":"system","content":"..."}
HumanMessageThe user's turn{"role":"user","content":"..."}
AIMessageA prior model reply (for history){"role":"assistant","content":"..."}
ToolMessageResult returned by a tool call{"role":"tool","content":"..."}

3.1 — Single-Turn Call

response = llm.invoke([
    HumanMessage(content="What is the capital of France?")
])
print(response.content)  # "The capital of France is Paris."

3.2 — With a System Prompt

response = llm.invoke([
    SystemMessage(content="You are a concise assistant. Reply in one sentence only."),
    HumanMessage(content="Explain what a neural network is.")
])
print(response.content)

3.3 — Multi-Turn (Simulated Memory via List)

The simplest form of memory — maintain a Python list and keep appending. Each call sends the full history.

history = [
    SystemMessage(content="You are a helpful Python tutor."),
    HumanMessage(content="What is a list comprehension?"),
]
r1 = llm.invoke(history)
print(r1.content)

# Extend history and continue
history.append(AIMessage(content=r1.content))
history.append(HumanMessage(content="Show me an example that filters even numbers."))
r2 = llm.invoke(history)
print(r2.content)

# Token usage grows with each turn
print(r2.usage_metadata)
04

Prompt Templates

Templates separate the prompt structure from the data. Reuse the same prompt shape with different variables.

4.1 — ChatPromptTemplate

from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are an expert in {domain}. Keep answers under 3 sentences."),
    ("user", "Explain: {topic}")
])

# Format into a message list
messages = prompt.format_messages(domain="machine learning", topic="gradient descent")
response = llm.invoke(messages)
print(response.content)

4.2 — Pipe Operator: prompt | model | parser

LangChain's LCEL (LangChain Expression Language) uses | to compose a chain. Each component's output is the next component's input.

from langchain_core.output_parsers import StrOutputParser

# Build the chain
chain = prompt | llm | StrOutputParser()

# Invoke — returns a plain string (not an AIMessage)
text: str = chain.invoke({"domain": "networking", "topic": "TCP handshake"})
print(text)

# Works for any variables
text2 = chain.invoke({"domain": "databases", "topic": "indexing"})
print(text2)
StrOutputParser strips the AIMessage wrapper and returns a plain str. Use it as the final step whenever downstream code expects a string.

Memory in LangChain

LangGraph's checkpointing system is the recommended memory approach in 2025+. Legacy ConversationBufferMemory is deprecated. There are three tiers.

TierClassPersists Across Restarts?Use Case
ManualPython listNoQuick demos, single-session scripts
In-MemoryMemorySaverNo (RAM only)Development, testing, Colab notebooks
PersistentSqliteSaver / PostgresSaverYesProduction apps, user sessions across restarts

5.1 — Python List (Manual)

No imports needed. Fine for notebooks. Limitation: you manage trimming to avoid token overflow.

from langchain_core.messages import SystemMessage, HumanMessage, AIMessage

history = [SystemMessage(content="You are a helpful tutor.")]

def chat(user_input: str) -> str:
    history.append(HumanMessage(content=user_input))
    response = llm.invoke(history)
    history.append(AIMessage(content=response.content))
    return response.content

print(chat("What is recursion?"))
print(chat("Give me a Python example of it."))
print(chat("What did I just ask you about?"))  # proves memory works

5.2 — MemorySaver (LangGraph Checkpointer)

Attach a MemorySaver to a compiled LangGraph graph. State is saved automatically after every step. Each conversation thread is identified by a thread_id.

from langchain.chat_models import init_chat_model
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver

llm = init_chat_model(f"google_genai:{GEMINI_MODEL}", api_key=GEMINI_API_KEY)

# MemorySaver stores state in RAM — fine for dev/Colab
checkpointer = MemorySaver()

# Compile agent with checkpointer — this is all you need
agent = create_react_agent(
    model=llm,
    tools=[],                # no tools for this demo
    checkpointer=checkpointer,
    prompt="You are a helpful assistant."
)
Key concept: thread_id identifies a conversation session. Same thread_id = same memory. Different thread_id = fresh conversation. This is how you isolate different users. 
# thread_id is passed at invocation time via config
config = {"configurable": {"thread_id": "user-alice-session-1"}}

def chat_with_memory(user_input: str, cfg: dict) -> str:
    result = agent.invoke(
        {"messages": [{"role": "user", "content": user_input}]},
        config=cfg
    )
    return result["messages"][-1].content

# These calls share memory because they use the same thread_id
print(chat_with_memory("My name is Alice.", config))
print(chat_with_memory("What is 7 × 8?", config))
print(chat_with_memory("What is my name?", config))  # "Your name is Alice."

# Fresh session — no memory of Alice
config2 = {"configurable": {"thread_id": "user-bob-session-1"}}
print(chat_with_memory("What is my name?", config2))  # "I don't know your name."

5.3 — SqliteSaver (Persistent)

Drop-in replacement for MemorySaver. Writes checkpoints to a .db file. Conversations survive restarts.

from langgraph.checkpoint.sqlite import SqliteSaver

# Context manager handles open/close of the DB connection
with SqliteSaver.from_conn_string("./checkpoints.db") as checkpointer:
    agent = create_react_agent(
        model=llm,
        tools=[],
        checkpointer=checkpointer,
    )
    config = {"configurable": {"thread_id": "persistent-session-1"}}
    result = agent.invoke(
        {"messages": [{"role": "user", "content": "Remember: my favourite colour is green."}]},
        config=config
    )
    print(result["messages"][-1].content)

# Restart script, reconnect — memory survives
with SqliteSaver.from_conn_string("./checkpoints.db") as checkpointer:
    agent = create_react_agent(model=llm, tools=[], checkpointer=checkpointer)
    result = agent.invoke(
        {"messages": [{"role": "user", "content": "What is my favourite colour?"}]},
        config={"configurable": {"thread_id": "persistent-session-1"}}
    )
    print(result["messages"][-1].content)  # "Your favourite colour is green."

5.4 — Interactive Conversation Loop

⟳ Conversation Loop

A real interactive loop. User types, agent responds, memory is automatic via checkpointer.

from langchain.chat_models import init_chat_model
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver
import uuid

llm = init_chat_model(f"google_genai:{GEMINI_MODEL}", api_key=GEMINI_API_KEY)
checkpointer = MemorySaver()
agent = create_react_agent(
    model=llm,
    tools=[],
    checkpointer=checkpointer,
    prompt="You are a friendly assistant with a good memory."
)

# Each session gets a unique thread_id
session_id = str(uuid.uuid4())
config = {"configurable": {"thread_id": session_id}}

print(f"Session started. Thread: {session_id}")
print("Type 'exit' to quit.\n")

while True:
    user_input = input("You: ").strip()
    if not user_input or user_input.lower() == "exit":
        print("Session ended.")
        break

    result = agent.invoke(
        {"messages": [{"role": "user", "content": user_input}]},
        config=config
    )
    reply = result["messages"][-1].content
    print(f"AI: {reply}\n")

RAG — Retrieval-Augmented Generation

RAG lets the model answer questions about your data (PDFs, docs, websites) without fine-tuning. Four steps: load → split → embed → retrieve → generate.

Documents → load → Chunks → embed → VectorStore → retrieve → Context + Prompt → LLM → Answer

6.1 — Document Loaders

Load raw content from various sources into Document objects.

from langchain_community.document_loaders import (
    TextLoader,
    PyPDFLoader,
    WebBaseLoader,
    DirectoryLoader,
    CSVLoader,
)

# Load a plain text file
loader = TextLoader("my_notes.txt")
docs = loader.load()  # list of Document objects

# Load a PDF (each page becomes a Document)
loader = PyPDFLoader("report.pdf")
docs = loader.load()

# Scrape a webpage
loader = WebBaseLoader("https://en.wikipedia.org/wiki/Artificial_intelligence")
docs = loader.load()

# Load all .txt files in a folder
loader = DirectoryLoader("./data/", glob="**/*.txt", loader_cls=TextLoader)
docs = loader.load()

print(f"Loaded {len(docs)} documents")
print(docs[0].page_content[:200])  # raw text
print(docs[0].metadata)            # source, page number, etc.

6.2 — Text Splitters

LLMs have context limits. Split documents into chunks before embedding. RecursiveCharacterTextSplitter is the default choice.

from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,       # characters per chunk
    chunk_overlap=50,     # overlap to avoid losing context at boundaries
    separators=["\n\n", "\n", " ", ""],  # tries each in order
)

chunks = splitter.split_documents(docs)

print(f"Split into {len(chunks)} chunks")
print(chunks[0].page_content)
print(chunks[0].metadata)  # inherits source metadata
chunk_overlap matters for retrieval quality. Overlapping windows ensure a sentence split across two chunks is still retrievable.

6.3 — Embeddings

Convert text into numeric vectors. Semantically similar text lands close together in vector space.

from langchain_google_genai import GoogleGenerativeAIEmbeddings

embeddings = GoogleGenerativeAIEmbeddings(
    model="models/text-embedding-004",
    google_api_key=GEMINI_API_KEY
)

# Embed a single string
vector = embeddings.embed_query("What is machine learning?")
print(f"Vector dimension: {len(vector)}")  # 768

# Embed a batch of documents (used internally by vectorstores)
vectors = embeddings.embed_documents(["text 1", "text 2", "text 3"])

6.4 — Vectorstores

Store vectors and support similarity search. Chroma is the standard local choice. FAISS is an alternative (pip install faiss-cpu).

from langchain_chroma import Chroma

# Create vectorstore from chunks (embeds automatically)
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./chroma_db",   # omit for in-memory
)

# Similarity search — returns top-k most relevant chunks
results = vectorstore.similarity_search("explain neural networks", k=3)
for doc in results:
    print(doc.page_content[:150])
    print("---")

# Load an existing persisted store
vectorstore = Chroma(
    persist_directory="./chroma_db",
    embedding_function=embeddings
)

# Use as a retriever (integrates with chains)
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

6.5 — RAG Conversation Chain

⟳ Conversation Loop

A full RAG chain with conversation memory. Uses checkpointer so context is maintained across turns.

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough
from langchain_google_genai import ChatGoogleGenerativeAI, GoogleGenerativeAIEmbeddings
from langchain_chroma import Chroma
from langchain_community.document_loaders import WebBaseLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

# --- 1. Load & index documents ---
loader = WebBaseLoader("https://en.wikipedia.org/wiki/Machine_learning")
docs = loader.load()
chunks = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50).split_documents(docs)
embeddings = GoogleGenerativeAIEmbeddings(
    model="models/text-embedding-004", google_api_key=GEMINI_API_KEY
)
vectorstore = Chroma.from_documents(chunks, embeddings)
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

# --- 2. Build the RAG chain ---
llm = ChatGoogleGenerativeAI(model=GEMINI_MODEL, api_key=GEMINI_API_KEY)

rag_prompt = ChatPromptTemplate.from_messages([
    ("system", "Answer using ONLY the context below. If it's not there, say so.\n\nContext:\n{context}"),
    ("human", "{question}"),
])

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | rag_prompt
    | llm
    | StrOutputParser()
)

# --- 3. Conversation loop ---
# Maintain a simple history list for the conversation context
history = []

print("RAG Assistant ready. Ask questions about the loaded document.\n")
while True:
    q = input("You: ").strip()
    if q.lower() == "exit":
        break
    answer = rag_chain.invoke(q)
    history.append({"question": q, "answer": answer})
    print(f"AI: {answer}\n")
07

Chains

Chains compose multiple steps using the | pipe operator (LCEL). They are stateless by default — memory must be added explicitly.

When to use chains vs agents: Chains are deterministic — you hard-code the steps. Agents are dynamic — the LLM decides which steps to take. Use chains when the workflow is fixed; agents when it needs to reason.

7.1 — Sequential Chain

Output of step A becomes input of step B. Common pattern: summarise → translate → format.

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_google_genai import ChatGoogleGenerativeAI

llm = ChatGoogleGenerativeAI(model=GEMINI_MODEL, api_key=GEMINI_API_KEY)
parser = StrOutputParser()

# Step 1: Summarise
summarise_prompt = ChatPromptTemplate.from_template(
    "Summarise the following text in 2 sentences:\n\n{text}"
)

# Step 2: Make it formal
formalise_prompt = ChatPromptTemplate.from_template(
    "Rewrite this in a formal, professional tone:\n\n{summary}"
)

# Build sequential chain — output of step 1 feeds {summary} in step 2
step1 = summarise_prompt | llm | parser
step2 = formalise_prompt | llm | parser

# Chain them — output of step1 is a string, fed as {"summary": ...} into step2
sequential_chain = step1 | (lambda summary: {"summary": summary}) | step2

result = sequential_chain.invoke({
    "text": "Machine learning is a way for computers to learn from data. "
            "Instead of being programmed with rules, the system finds patterns by itself."
})
print(result)

7.2 — Parallel Chain (RunnableParallel)

Run multiple chains simultaneously on the same input. Results are combined into a dict.

from langchain_core.runnables import RunnableParallel

pros_prompt   = ChatPromptTemplate.from_template("List 3 pros of {technology}. Be brief.")
cons_prompt   = ChatPromptTemplate.from_template("List 3 cons of {technology}. Be brief.")
future_prompt = ChatPromptTemplate.from_template("In one sentence, future of {technology}?")

# All three run in parallel on the same input
analysis_chain = RunnableParallel(
    pros   = pros_prompt   | llm | parser,
    cons   = cons_prompt   | llm | parser,
    future = future_prompt | llm | parser,
)

result = analysis_chain.invoke({"technology": "blockchain"})
print("PROS:\n",   result["pros"])
print("CONS:\n",   result["cons"])
print("FUTURE:\n", result["future"])

7.3 — Branched Chain (RunnableBranch)

Route to different chains based on a condition. Acts like an if/elif/else for chains.

from langchain_core.runnables import RunnableBranch, RunnableLambda

# Detect intent (simple keyword classifier)
def detect_intent(x: dict) -> dict:
    text = x["question"].lower()
    if any(w in text for w in ["code", "python", "function", "class"]):
        return {**x, "intent": "code"}
    elif any(w in text for w in ["explain", "what is", "define", "meaning"]):
        return {**x, "intent": "explain"}
    else:
        return {**x, "intent": "general"}

code_chain    = ChatPromptTemplate.from_template("Write clean Python code for: {question}") | llm | parser
explain_chain = ChatPromptTemplate.from_template("Explain clearly for a beginner: {question}") | llm | parser
general_chain = ChatPromptTemplate.from_template("Answer helpfully: {question}") | llm | parser

branch = RunnableBranch(
    (lambda x: x["intent"] == "code",    code_chain),
    (lambda x: x["intent"] == "explain", explain_chain),
    general_chain  # default fallback
)

full_chain = RunnableLambda(detect_intent) | branch

print(full_chain.invoke({"question": "write a Python function to reverse a string"}))
print(full_chain.invoke({"question": "what is backpropagation"}))
print(full_chain.invoke({"question": "best pizza places in Delhi"}))

7.4 — Conversation Loop with Memory

⟳ Conversation Loop

A chain-based conversation that maintains history across turns. Since chains are stateless, we pass history explicitly each call.

from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.messages import HumanMessage, AIMessage
from langchain_core.output_parsers import StrOutputParser
from langchain_google_genai import ChatGoogleGenerativeAI

llm = ChatGoogleGenerativeAI(model=GEMINI_MODEL, api_key=GEMINI_API_KEY)

# MessagesPlaceholder injects the history list directly into the prompt
chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a knowledgeable assistant. Remember what the user says."),
    MessagesPlaceholder(variable_name="history"),
    ("human", "{input}"),
])

chain = chat_prompt | llm | StrOutputParser()

# Persistent history list
history = []

def chat(user_input: str) -> str:
    response = chain.invoke({"input": user_input, "history": history})
    # Update history AFTER invocation
    history.append(HumanMessage(content=user_input))
    history.append(AIMessage(content=response))
    return response

# Interactive loop
print("Chain Chat (type 'exit' to quit)\n")
while True:
    user_input = input("You: ").strip()
    if user_input.lower() == "exit":
        break
    if user_input:
        print(f"AI: {chat(user_input)}\n")

7.5 — Limitations of Chains → Enter Agents

  • Chains require you to decide the steps upfront — no dynamic reasoning
  • Cannot decide whether to call a tool or which tool to call
  • Cannot retry, self-correct, or loop until a goal is reached
  • No native state persistence (must wire memory manually)
  • Agents solve all of this — the LLM reasons about what to do next
08

Agents

An agent is an LLM that can call tools, observe results, and decide what to do next — in a loop — until it has a final answer. LangChain agents are compiled LangGraph graphs under the hood.

User Message
↓
Agent (LLM) ← reasons: do I need a tool?
↓ (if tool needed)
Tool Execution ← runs the function
↓ ToolMessage back
Agent (LLM) ← observes result, decides next
↓ (loop until done)
Final Answer

8.1 — create_react_agent (Core API)

The primary way to create a ReAct agent with LangGraph. Every other agent type builds on this.

from langchain.chat_models import init_chat_model
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver

llm = init_chat_model(f"google_genai:{GEMINI_MODEL}", api_key=GEMINI_API_KEY)
checkpointer = MemorySaver()

agent = create_react_agent(
    model=llm,
    tools=[],                     # tool list — empty for now
    checkpointer=checkpointer,    # enables multi-turn memory
    prompt="You are a helpful AI assistant.",  # system instruction
    # name="my-agent",            # optional: for multi-agent systems
)

# Invoke
config = {"configurable": {"thread_id": "demo-1"}}
result = agent.invoke(
    {"messages": [{"role": "user", "content": "Hello! What can you do?"}]},
    config=config
)
print(result["messages"][-1].content)

# Inspect the full message trace
for msg in result["messages"]:
    print(f"[{type(msg).__name__}] {msg.content[:80]}")
Unified API (LangChain 0.3+): You may also see from langchain.agents import create_agent — this is a higher-level wrapper that delegates to create_react_agent. Both accept the same parameters.

8.2 — Defining Tools with @tool

Tools are Python functions with a descriptive docstring. The LLM reads the docstring to decide when and how to call the tool.

from langchain_core.tools import tool
from datetime import datetime

# --- Basic synchronous tool ---
@tool
def calculate_area(length: float, width: float) -> str:
    """Calculate the area of a rectangle.
    Use when the user asks about rectangular area given length and width.
    Both inputs must be non-negative numbers in metres."""
    if length < 0 or width < 0:
        return "Error: dimensions must be non-negative."
    return f"Area = {length * width:.2f} m²"

# --- Tool with custom name ---
@tool("get-time", description="Returns current time in the given timezone (e.g. 'Asia/Kolkata').")
def get_current_time(timezone: str = "UTC") -> str:
    try:
        from pytz import timezone as tz
        return datetime.now(tz(timezone)).strftime("%Y-%m-%d %H:%M:%S %Z")
    except Exception as e:
        return f"Error: {e}"

# --- Async tool (preferred for I/O bound operations) ---
@tool
async def fetch_weather(city: str) -> str:
    """Fetch current weather for a given city.
    Use when user asks about weather, temperature, or conditions."""
    # In real code, call a weather API here
    return f"Weather in {city}: 28°C, partly cloudy"

# --- Test tools directly ---
print(calculate_area.invoke({"length": 10.5, "width": 4.0}))
print(get_current_time.invoke({"timezone": "Asia/Kolkata"}))

# Tool metadata
print(calculate_area.name)         # "calculate_area"
print(calculate_area.description)  # the docstring
print(calculate_area.args)         # {"length": ..., "width": ...}
Docstring quality = tool call accuracy. Write specific, clear docstrings. Say what the tool does, when to use it, and what format inputs should be in. Vague docstrings lead to the agent calling the wrong tool.

8.3 — RunnableConfig & Per-Call Configuration

RunnableConfig is a dict passed at runtime — not baked into the agent. It controls thread identity, recursion limits, callbacks, and more.

from langchain_core.runnables import RunnableConfig

# Minimal: just the thread_id for memory
config: RunnableConfig = {
    "configurable": {
        "thread_id": "user-123",
    }
}

# Full options
config: RunnableConfig = {
    "configurable": {
        "thread_id": "user-123",           # identifies the memory thread
    },
    "recursion_limit": 25,                 # max agent loop iterations (default 25)

8.4 — Store Object for Cross-Session Memory

InMemoryStore adds a namespace-aware key-value store to the agent. Unlike checkpointer (which stores per-thread state), the store persists across threads — e.g. user preferences that survive a new conversation.

from langgraph.store.memory import InMemoryStore
from langgraph.prebuilt import create_react_agent
from langchain_core.tools import tool

# In-memory store (replace with AsyncPostgresStore for production)
store = InMemoryStore()

# Tools can read/write to the store
@tool
def save_preference(key: str, value: str) -> str:
    """Save a user preference so it can be recalled later.
    Example: save_preference('favourite_color', 'blue')"""
    store.put(("user-prefs", "alice"), key, {"value": value})
    return f"Saved: {key} = {value}"

@tool
def get_preference(key: str) -> str:
    """Retrieve a previously saved user preference by key."""
    result = store.get(("user-prefs", "alice"), key)
    if result:
        return f"{key} = {result.value['value']}"
    return f"No preference found for '{key}'"

from langgraph.checkpoint.memory import MemorySaver
from langchain.chat_models import init_chat_model

llm = init_chat_model(f"google_genai:{GEMINI_MODEL}", api_key=GEMINI_API_KEY)

agent = create_react_agent(
    model=llm,
    tools=[save_preference, get_preference],
    checkpointer=MemorySaver(),   # per-thread state
    store=store,                  # cross-thread long-term memory
    prompt="You are a personalisation assistant. Save and recall user preferences."
)

config_t1 = {"configurable": {"thread_id": "thread-1"}}
agent.invoke({"messages": [{"role": "user", "content": "My favourite colour is teal."}]}, config=config_t1)

# New thread — checkpointer knows nothing about thread-1
# But the store retains the preference
config_t2 = {"configurable": {"thread_id": "thread-2"}}
result = agent.invoke(
    {"messages": [{"role": "user", "content": "What is my favourite colour?"}]},
    config=config_t2
)
print(result["messages"][-1].content)  # recalls "teal" via store

8.5 — Human-in-the-Loop (Interrupts)

Interrupts pause the agent before or after a tool call, letting a human inspect/approve the action. Built into LangGraph.

from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver
from langchain_core.tools import tool
from langchain.chat_models import init_chat_model

llm = init_chat_model(f"google_genai:{GEMINI_MODEL}", api_key=GEMINI_API_KEY)

@tool
def send_email(to: str, subject: str, body: str) -> str:
    """Send an email to a recipient. Use when user explicitly asks to send an email."""
    # Real implementation would call an email API
    return f"Email sent to {to}: '{subject}'"

@tool
def get_contacts() -> str:
    """Return a list of available contacts."""
    return "Contacts: alice@example.com, bob@example.com"

checkpointer = MemorySaver()

# interrupt_before=["tools"] pauses execution BEFORE any tool runs
agent = create_react_agent(
    model=llm,
    tools=[send_email, get_contacts],
    checkpointer=checkpointer,
    interrupt_before=["tools"],   # pause before tool execution for human approval
)

config = {"configurable": {"thread_id": "hitl-demo"}}

# Step 1: Stream until the interrupt
events = agent.stream(
    {"messages": [{"role": "user", "content": "Send an email to alice@example.com saying hello."}]},
    config=config
)

state = None
for event in events:
    if "__interrupt__" in event:
        interrupt_info = event["__interrupt__"]
        print("⚠️  Agent wants to call a tool. Review:")
        print(interrupt_info)

# Step 2: Human decision — approve or reject
approval = input("Approve tool call? (yes/no): ").strip().lower()

if approval == "yes":
    # Resume the agent — None means "proceed as planned"
    result = agent.invoke(None, config=config)
    print(result["messages"][-1].content)
else:
    # Inject a rejection message and resume
    from langchain_core.messages import HumanMessage
    agent.invoke(
        {"messages": [HumanMessage(content="User rejected the tool call. Explain what you were going to do instead.")]},
        config=config
    )

8.6 — Full Agent Conversation Loop

⟳ Conversation Loop

A complete, production-style agent with tools, memory via checkpointer, and an interactive conversation loop.

import asyncio
import uuid
import wikipedia
import platform
from datetime import datetime
from langchain.chat_models import init_chat_model
from langchain_core.tools import tool
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver

# ── Tools ──────────────────────────────────────────────────
@tool
def calculate_area(length: float, width: float) -> str:
    """Calculate the area of a rectangle. Inputs: length and width as floats."""
    return f"Area = {length * width:.2f} sq units"

@tool
def search_wikipedia(query: str) -> str:
    """Search Wikipedia and return a 3-sentence summary of the topic.
    Use for factual questions, definitions, or historical information."""
    try:
        return wikipedia.summary(query, sentences=3)
    except wikipedia.exceptions.PageError:
        return f"No Wikipedia page found for '{query}'."
    except wikipedia.exceptions.DisambiguationError as e:
        return f"Ambiguous. Options: {', '.join(e.options[:5])}"

@tool
def get_current_time(timezone: str = "UTC") -> str:
    """Return current date and time in the given timezone.
    Use when user asks about the current time or date."""
    try:
        from pytz import timezone as tz
        return datetime.now(tz(timezone)).strftime("%Y-%m-%d %H:%M:%S %Z")
    except Exception as e:
        return f"Error: {e}"

@tool
def get_system_info() -> str:
    """Return information about the current system environment.
    Use when user asks about the machine, OS, Python version, etc."""
    return (f"OS: {platform.system()} {platform.version()} | "
            f"Arch: {platform.machine()} | Python: {platform.python_version()}")

# ── Agent Setup ────────────────────────────────────────────
llm = init_chat_model(f"google_genai:{GEMINI_MODEL}", api_key=GEMINI_API_KEY)
checkpointer = MemorySaver()

agent = create_react_agent(
    model=llm,
    tools=[calculate_area, search_wikipedia, get_current_time, get_system_info],
    checkpointer=checkpointer,
    prompt=(
        "You are a helpful AI assistant with access to tools. "
        "Always use the appropriate tool when the user's request requires it. "
        "Maintain context from the conversation history."
    )
)

# ── Conversation Loop ──────────────────────────────────────
async def run_agent_loop():
    session_id = str(uuid.uuid4())
    config = {"configurable": {"thread_id": session_id}}

    print(f"Agent ready | Session: {session_id[:8]}...")
    print("Tools: calculate_area, search_wikipedia, get_current_time, get_system_info")
    print("Type 'exit' to quit, 'history' to see message count.\n")

    while True:
        user_input = input("You: ").strip()
        if not user_input:
            continue
        if user_input.lower() == "exit":
            print("Session ended.")
            break
        if user_input.lower() == "history":
            state = agent.get_state(config)
            print(f"  Message count: {len(state.values.get('messages', []))}\n")
            continue

        result = await agent.ainvoke(
            {"messages": [{"role": "user", "content": user_input}]},
            config=config
        )

        final_msg = result["messages"][-1]
        print(f"AI: {final_msg.content}\n")

# Run the loop
await run_agent_loop()   # use asyncio.run(run_agent_loop()) outside of notebooks

Multi-Agent Systems

When tasks are too complex for one agent, split them across specialist agents. A supervisor receives the user's request and delegates to the right sub-agent.

User
↓
Orchestrator / Supervisor ← the only agent that talks to user
↓ delegates to
Specialist Agent A (e.g. Research Agent)
Specialist Agent B (e.g. Code Agent)
Specialist Agent C (e.g. Summariser Agent)
↓ results back to Orchestrator
Orchestrator ← synthesises and replies to user

Pattern 1 — Tool-as-Agent

The simplest pattern. Wrap a specialist agent as a tool that the orchestrator calls.

from langchain.chat_models import init_chat_model
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver
from langchain_core.tools import tool

llm = init_chat_model(f"google_genai:{GEMINI_MODEL}", api_key=GEMINI_API_KEY)

# ── Specialist: Research Agent ────────────────────────────
import wikipedia

@tool
def wiki_search(query: str) -> str:
    """Search Wikipedia."""
    try:
        return wikipedia.summary(query, sentences=5)
    except Exception as e:
        return str(e)

research_agent = create_react_agent(
    model=llm,
    tools=[wiki_search],
    prompt="You are a research specialist. Find accurate factual information."
)

# ── Wrap the specialist as a tool for the orchestrator ────
@tool
def research(query: str) -> str:
    """Delegate a factual research question to the research specialist agent.
    Use when the user asks about facts, history, science, or definitions."""
    result = research_agent.invoke({
        "messages": [{"role": "user", "content": query}]
    })
    return result["messages"][-1].content

# ── Orchestrator ──────────────────────────────────────────
orchestrator = create_react_agent(
    model=llm,
    tools=[research],
    checkpointer=MemorySaver(),
    prompt=(
        "You are the main orchestrator. Delegate research tasks to the research tool. "
        "Synthesise results and reply to the user in a clear, friendly manner."
    )
)

config = {"configurable": {"thread_id": "multi-agent-demo"}}
result = orchestrator.invoke(
    {"messages": [{"role": "user", "content": "Tell me about quantum computing."}]},
    config=config
)
print(result["messages"][-1].content)

from langgraph_supervisor import create_supervisor
from langgraph.prebuilt import create_react_agent
from langchain.chat_models import init_chat_model
from langchain_core.tools import tool

llm = init_chat_model(f"google_genai:{GEMINI_MODEL}", api_key=GEMINI_API_KEY)

# Specialist agents
@tool
def python_exec(code: str) -> str:
    """Execute Python code and return the output. Use for maths and code tasks."""
    import io, contextlib
    buf = io.StringIO()
    try:
        with contextlib.redirect_stdout(buf):
            exec(code, {})
        return buf.getvalue() or "Executed. No output."
    except Exception as e:
        return f"Error: {e}"

code_agent = create_react_agent(llm, tools=[python_exec], name="code_agent",
    prompt="You are a Python coding specialist. Write and execute code.")

research_agent = create_react_agent(llm, tools=[], name="research_agent",
    prompt="You are a research specialist. Answer factual questions thoroughly.")

# Supervisor routes between specialists
supervisor = create_supervisor(
    agents=[code_agent, research_agent],
    model=llm,
    prompt=(
        "You are the supervisor. Route coding/math questions to code_agent. "
        "Route research/factual questions to research_agent. "
        "Synthesise results and reply to the user."
    )
).compile()

result = supervisor.invoke({
    "messages": [{"role": "user", "content": "What is 15 factorial? Also, who invented Python?"}]
})
print(result["messages"][-1].content)

Key Principles for Multi-Agent Design

  • Only the orchestrator communicates with the user — specialists never talk directly
  • Each specialist has a narrow, well-defined set of tools and a focused system prompt
  • Give each agent a name= parameter — it appears in LangSmith traces for debugging
  • The orchestrator holds the checkpointer for conversation memory — specialists are usually stateless
  • Use RunnableConfig metadata to pass user context (user_id, permissions) to all agents

Memory Decision Tree

1
Need memory across messages in the same session? → Any checkpointer with same thread_id
2
Need memory to survive process restart? → SqliteSaver or PostgresSaver
3
Need memory to cross session boundaries (user prefs, facts about user)? → store=InMemoryStore() or AsyncPostgresStore
4
Multi-user app? → Generate a unique thread_id per user session (str(uuid.uuid4()))
